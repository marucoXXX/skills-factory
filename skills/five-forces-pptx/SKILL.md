# Five Forces PowerPoint ジェネレーター

マイケル・ポーターが提唱した**Five Forces（5つの競争要因）**フレームワークを使って業界の競争構造を分析し、PowerPointスライドを生成するスキル。

---

## Five Forcesフレームワークとは

業界の収益性に影響を与える5つの競争要因（脅威・力）を分析し、業界構造と自社の競争優位性を評価する外部環境分析フレームワーク。ポーターは「競争戦略とは業界に働く5つの競争要因からうまく自社を守り、自社に有利になるように競争要因を動かせる位置を業界内に見つけること」と定義した。

| 要素 | 定義 | 分析の観点 |
|------|------|------|
| **業界内の競争** | 同業他社との競合状況の強さ | 競合数、規模の均衡度、差別化の程度、成長率、固定費構造、撤退障壁 |
| **新規参入の脅威** | 新たなプレイヤーが参入する可能性と容易さ | 参入障壁の高さ（規模の経済、ブランド、規制、資本要件、スイッチングコスト） |
| **代替品の脅威** | 異なる手段で同じニーズを満たす製品・サービスの存在 | 代替品の品質・価格比、スイッチングコスト、顧客の代替意向 |
| **売り手の交渉力** | 供給業者（サプライヤー）が価格・品質・条件で圧力をかける力 | サプライヤーの集中度、差別化度、スイッチングコスト、代替供給源の有無 |
| **買い手の交渉力** | 顧客が値下げ・品質向上を要求する力 | 顧客の集中度、購買量、製品の差別化度、スイッチングコスト、情報の非対称性 |

### スライド要素の定義

| 要素 | 定義 | ポイント |
|------|------|------|
| **Main Message** | Five Forces分析全体を総括し、読み手にアクションを促す一文。最大70文字 | 必ず「〜すべき」で締める |
| **Chart Title** | スライドのサブタイトル。分析対象の業界・文脈を端的に示す短いフレーズ（10〜20文字） | Main Messageを補足する文脈を示す |

### 記述ルール

- **Main Message**: 最大70文字。Five Forces分析全体を一言で総括し、必ず「〜すべき」で終える。ユーザーが指定した場合はそのまま使用、指定がない場合は分析内容をもとにドラフトして確認を取る。
- **Chart Title**: 10〜20文字程度。Main Messageのテーマを補足する文脈フレーズ。ユーザーが指定しない場合はドラフトして確認を取る。

### 分析のポイント

- **業界の定義が重要**: 分析対象の「業界」を明確に定義する。広すぎると焦点がぼやけ、狭すぎると重要な競争要因を見落とす
- **縦の3要素**（新規参入・業界内競争・代替品）は**売上**に直接影響する
- **横の2要素**（売り手・買い手の交渉力）は**利益**に直接影響する
- 各要素について「脅威が強い（収益性を圧迫）」か「脅威が弱い（収益機会がある）」かを評価する
- 最も影響力の大きい要素を重点的に分析し、そこから意味合い（戦略示唆）を導出する

## 入力パターンと処理フロー

### パターンA：議事録・調査メモが入力された場合

ユーザーから生のテキスト（議事録、調査レポート、ヒアリングメモなど）が渡された場合、あなたが分析を行う。

**いきなりPowerPointを作成しない**

1. **Step 1: 業界の定義を確認する** — どの業界を分析対象とするか、ユーザーに確認する（自明な場合はスキップ可）
2. **Step 2: Five Forces分析を実施し、Markdownで提示する**（後述の分析ガイドラインに従う）
3. **Step 3: ユーザーの承認後**、PowerPointを生成する

### パターンB：Five Forces分析結果が入力された場合

ユーザーが既に5つの要素ごとに整理済みのテキストを渡した場合。

1. **Step 1: 内容を確認**し、5要素が揃っているか、各要素に具体的なポイントがあるか確認
2. 不足・曖昧な点があれば補足提案を行う
3. **Step 2: 確認後、PowerPointを生成する**

## Step 1-2: Five Forces分析の実施と提示

### 議事録・メモからの分析ガイドライン

#### 業界内の競争

以下のような情報を探す：
- 「競合は〜」「シェアは〜」「価格競争が〜」「差別化が〜」
- 主要プレイヤーの名前、数、規模
- 市場成長率、利益率の動向

#### 新規参入の脅威

以下のような情報を探す：
- 「参入障壁が〜」「新規プレイヤーが〜」「規制が〜」
- 必要資本、技術的ハードル、ブランド力
- 最近の参入事例

#### 代替品の脅威

以下のような情報を探す：
- 「代替手段として〜」「〜に置き換わる可能性」
- 他業界からの製品・サービス
- テクノロジーの変化による代替リスク

#### 売り手の交渉力

以下のような情報を探す：
- 「サプライヤーが〜」「調達コストが〜」「仕入先が〜」
- サプライヤーの数と集中度
- 原材料・部品の希少性

#### 買い手の交渉力

以下のような情報を探す：
- 「顧客が〜」「価格交渉で〜」「コモディティ化が〜」
- 主要顧客の集中度
- スイッチングコストの高低

### 意味合いの導出

5つの要素を分析した後、**戦略上の意味合い（Implications）**を3つ程度導出する。意味合いとは、分析結果から導かれる「だから、こうすべき/こう考えるべき」という戦略示唆。

- 各フォースの強弱を総合的に判断し、業界の収益性への影響を評価する
- 自社にとって最もリスクの大きい/機会の大きい要素を特定する
- 具体的なアクションや方向性を示唆する

### Markdown出力フォーマット

ユーザーに確認を求める際は以下のフォーマットで出力する：

```markdown
## Five Forces分析結果

**Main Message**
〜すべき（最大70文字。必ず「〜すべき」で締める）

〜（スライドのサブタイトル。10〜20文字）

- ポイント1
- ポイント2

### 新規参入の脅威
- ポイント1
- ポイント2

### 代替品の脅威
- ポイント1
- ポイント2

### 売り手の交渉力
- ポイント1
- ポイント2
- ポイント3

### 買い手の交渉力
```

確認メッセージ例：
> 上記のFive Forces分析でよろしいでしょうか？修正があればお知らせください。確認後にPowerPointを生成します。

## Step 3: PowerPointの生成

### テンプレートの参照

テンプレートは `assets/five-forces-template.pptx` を使用する。

```bash
TEMPLATE="<skill-dir>/assets/five-forces-template.pptx"
```

テンプレートの構造は `references/template-mapping.md` を参照すること。初めて使う場合は以下で確認：

```bash
pip install "markitdown[pptx]" -q --break-system-packages
python -m markitdown <skill-dir>/assets/five-forces-template.pptx
```

### Five ForcesデータのJSON化

分析内容を `/home/claude/five_forces_data.json` に以下の形式で保存する：

```json
{
  "main_message": "〜すべき（最大70文字、必ず「〜すべき」で締める）",
  "chart_title": "〜（10〜20文字のサブタイトル）",
  "industry_competition": [
    "ポイント1",
    "ポイント2"
  ],
  "new_entrants": [
    "ポイント1",
    "ポイント2"
  ],
  "substitutes": [
    "ポイント1",
    "ポイント2"
  ],
  "supplier_power": [
    "ポイント1",
    "ポイント2",
    "ポイント3"
  ],
  "buyer_power": [
    "ポイント1",
    "ポイント2",
    "ポイント3"
  ],
  "implications": [
    "意味合い1",
    "意味合い2",
    "意味合い3"
  ]
}
```

各要素のポイント数は柔軟に設定可能（2〜4つ程度が推奨）。Five Forcesボックスの各ポイントは30文字以内を目安にする（ボックスが小さいため）。意味合いのテキストボックスは折り返し表示に対応しているため、1ポイントあたり60文字程度まで許容される。

### スクリプト実行コマンド

```bash
python <skill-dir>/scripts/fill_five_forces.py \
  --data /home/claude/five_forces_data.json \
  --template <skill-dir>/assets/five-forces-template.pptx \
  --output /mnt/user-data/outputs/FiveForces_output.pptx
```

### 出力確認

```bash
python -m markitdown /mnt/user-data/outputs/FiveForces_output.pptx
```

内容が正しく反映されているか確認し、ユーザーに提示する。

## 品質チェックリスト

PowerPoint生成後、以下を確認：

- [ ] Main Messageが70文字以内で「〜すべき」で終わっているか
- [ ] Chart Titleが10〜20文字程度で文脈を示しているか
- [ ] 5つの要素すべてにポイントが記載されているか
- [ ] 意味合いが3つ程度あり、分析から論理的に導かれているか
- [ ] PPTXのmarkitdown出力でプレースホルダー（Competitor1, Buyer1等）が残っていないか
- [ ] Five Forcesボックスの各ポイントが30文字以内に収まっているか
- [ ] ドラフトしたMain Message/Chart Titleに「※ドラフト」が付いているか
