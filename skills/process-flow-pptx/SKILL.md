# プロセスフロー PowerPoint ジェネレーター

業務プロセスや推進ステップを**フロー図＋プロセス特徴**の1枚スライドにまとめる、戦略コンサルタント品質のPowerPointを生成するスキル。

---

## プロセスフローとは

プロセスフロースライドは、業務の全体像を**時系列のステップ（左側）**と**プロセスの特徴・ポイント（右側）**で構造化し、1枚で俯瞰できるようにするコンサル定番フォーマット。

### スライド構成

| 領域 | 内容 |
|------|------|
| **Main Message** | スライド全体を総括し、読み手にアクションを促す一文。最大70文字。必ず「〜すべき」で締める |
| **Chart Title** | スライドのサブタイトル。プロセスのテーマや文脈を端的に示す短いフレーズ（10〜20文字） |
| **プロセスステップ（左側）** | 3〜9ステップの矢印付きフロー図（3列×1〜3行） |
| **プロセス特徴（右側）** | 3つの特徴ラベルと、それぞれ2つのコメント（補足説明） |

### ステップ数の柔軟対応

テンプレートは **ProcessFlow9.pptx** の1つだけを使用する。ステップ数が9未満の場合、不要なプロセスボックスと矢印コネクタをスクリプトが自動削除する。

| ステップ数 | レイアウト | 削除されるShape |
|---|---|---|
| 9 | 3列×3行 | なし |
| 7〜8 | 3列×3行（3行目が1〜2個） | 不要なボックス＋コネクタ |
| 4〜6 | 3列×2行（2行目が1〜3個） | Row3全体 ＋ 不要なボックス＋コネクタ |
| 3 | 3列×1行 | Row2・Row3全体 |

## 記述ルール

### Main Message
- 最大70文字
- スライド全体を一言で総括し、必ず「〜すべき」で終える
- ユーザーが指定した場合はそのまま使用、指定がない場合はドラフトして確認を取る

### Chart Title
- 10〜20文字程度
- Main Messageのテーマを補足する文脈フレーズ
- ユーザーが指定しない場合はドラフトして確認を取る

### プロセスステップ
- 各ステップは**短いラベル**（例: 「現状分析」「要件定義」「設計」）
- 1ステップあたり10文字以内を目安とする
- 時系列順に左から右、上から下へ並べる

### プロセス特徴
- **3つの特徴**を記述する（Rule of Three）
- 各特徴は**ラベル**（太字）＋**2つのコメント**（補足説明）で構成
- ラベルは短くインパクトのある表現（例: 「段階的導入」「データ駆動」）
- コメントはラベルを補足する具体的な説明（各30〜50文字程度）

## 入力パターンと処理フロー

### パターンA：議事録・文字起こしが入力された場合

**いきなりPowerPointを作成しない**

1. **Step 1: プロセスフローのドラフトを作成する**（後述の抽出ガイドラインに従う）
2. **Step 2: Markdownでユーザーに提示**し、確認・修正を求める
3. **Step 3: ユーザーの承認後**、PowerPointを生成する

### パターンB：プロセスフロー形式のテキストが入力された場合

1. **Step 1: 内容を確認**し、ステップ数（3〜9）・特徴が揃っているか確認
2. 不足・曖昧な点があれば修正提案を行う
3. **Step 2: 確認後、PowerPointを生成する**

---

## Step 1: 議事録からプロセスフローを抽出する

### プロセスステップの抽出

議事録から**時系列の手順・フェーズ**を拾う。以下のような表現を探す：
- 「まず〜」「次に〜」「その後〜」「最後に〜」
- 「Phase 1」「Step 1」「第1段階」
- 「〜してから〜する」「〜の後に〜を行う」

**ポイント**:
- ステップは3〜9個に収める
- ステップが多すぎる場合は統合・抽象化して収める
- ステップが少なすぎる場合は分解して最低3個にする

### プロセス特徴の抽出

議事録から**プロセスのポイント・特徴・こだわり**を拾う：
- 「特徴として〜」「ポイントは〜」「重要なのは〜」
- 「〜を重視する」「〜がカギとなる」
- プロセス全体に関わる方針・原則・設計思想

**明示されていない場合は、戦略コンサルタントとして論理的にドラフトする**。ドラフトした場合は「※ドラフト」と明記する。

## Step 2: プロセスフローのMarkdown出力フォーマット

ユーザーに確認を求める際は、以下のフォーマットで出力する：

```markdown
## プロセスフロー整理結果

**Main Message（※ドラフト）**
〜すべき（最大70文字）

**Chart Title（※ドラフト）**
〜のテーマ（10〜20文字）

**ステップ数**: N ステップ

### プロセスステップ
1. ステップ名1
2. ステップ名2
3. ステップ名3
...

### プロセスの特徴
**特徴1：ラベル**
- コメント1
- コメント2

**特徴2：ラベル**
- コメント1
- コメント2

**特徴3：ラベル**
- コメント1
- コメント2
```

確認メッセージ例：
> 上記のプロセスフロー整理でよろしいでしょうか？Main Message、Chart Title、ステップ名、特徴も含めて修正があればお知らせください。確認後にPowerPointを生成します。

## Step 3: PowerPointの生成

### テンプレートの参照

テンプレートは `assets/ProcessFlow9.pptx` を使用する。ステップ数に応じて不要なShapeはスクリプトが自動削除する。

```bash
TEMPLATE="/mnt/skills/organization/process-flow-pptx/assets/ProcessFlow9.pptx"
```

テンプレートを初めて使う前に、以下で構造を確認すること：

```bash
pip install "markitdown[pptx]" -q --break-system-packages
python -m markitdown /mnt/skills/organization/process-flow-pptx/assets/ProcessFlow9.pptx
```

テンプレートのShape名は `references/template-mapping.md` に確認済みの対応表を記載している。

### プロセスフローデータのJSON化

内容を `/home/claude/process_flow_data.json` に以下の形式で保存する：

```json
{
  "main_message": "〜すべき（最大70文字）",
  "chart_title": "〜のテーマ（10〜20文字）",
  "steps": [
    "ステップ名1",
    "ステップ名2",
    "ステップ名3"
  ],
  "features": [
    {
      "label": "特徴ラベル1",
      "comment1": "コメント1の文章",
      "comment2": "コメント2の文章"
    },
    {
      "label": "特徴ラベル2",
      "comment1": "コメント1の文章",
      "comment2": "コメント2の文章"
    },
    {
      "label": "特徴ラベル3",
      "comment1": "コメント1の文章",
      "comment2": "コメント2の文章"
    }
  ]
}
```

- `steps`: 3〜9個のステップ名を配列で指定。スクリプトが個数に応じて不要Shapeを自動削除する

### スクリプト実行コマンド

```bash
python /mnt/skills/organization/process-flow-pptx/scripts/fill_process_flow.py \
  --data /home/claude/process_flow_data.json \
  --output /mnt/user-data/outputs/ProcessFlow_output.pptx
```

### 出力確認

```bash
python -m markitdown /mnt/user-data/outputs/ProcessFlow_output.pptx
```

内容が正しく反映されているか確認し、ユーザーに提示する。

## 品質チェックリスト

PowerPoint生成後、以下を確認：

- [ ] Main Messageが70文字以内で「〜すべき」で終わっているか
- [ ] Chart Titleが10〜20文字程度で文脈を的確に示しているか
- [ ] ステップ数が3〜9の範囲で、時系列順になっているか
- [ ] ステップ名が簡潔（10文字以内目安）か
- [ ] プロセス特徴が3つあり、各2コメント付きか
- [ ] 特徴ラベルが短くインパクトのある表現になっているか
- [ ] PPTXのmarkitdown出力で不要なプレースホルダーが残っていないか
