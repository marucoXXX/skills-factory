# プロセスフロー記述の良い例・悪い例

## テーマ：DXプラットフォーム導入プロセス

### ✅ 良い例（6ステップ）

**Main Message**
全社DXを加速するため、6段階の導入プロセスに沿って段階的にプラットフォームを展開すべき

**Chart Title**
DXプラットフォーム導入ロードマップ

**プロセスステップ**
1. 現状診断
2. 要件定義
3. PoC実施
4. 基盤構築
5. パイロット展開
6. 全社展開

**プロセス特徴**
- 段階的導入：PoCで仮説を検証してからパイロット展開に進み、リスクを最小化する / 各フェーズにGo/No-Goゲートを設け、経営判断を挟む設計とする
- データ駆動：現状診断で定量データを収集し、KPIベースで効果測定を行う / PoC段階でROIシミュレーションを実施し、投資対効果を事前に検証する
- 変革支援：パイロット部門でチェンジマネジメント施策を先行実施する / 全社展開時にはアンバサダー制度を導入し、現場の定着を促進する

---

### ❌ 悪い例（よくある間違い）

**Main Message（NG: 「〜すべき」で終わっていない、抽象的）**
DXを推進する

**Chart Title（NG: 長すぎる）**
当社のデジタルトランスフォーメーション推進に向けたプラットフォーム導入の全体プロセスと特徴について

**プロセスステップ（NG: 粒度がバラバラ、長すぎる）**
1. 現状の業務プロセスを分析して課題を特定する ← 長すぎる
2. 設計 ← 「要件定義」と「設計」は別だが一緒にできる粒度
3. 開発・テスト・導入・運用 ← 複数ステップを1つに詰め込みすぎ
4. 振り返り ← 他ステップと粒度が合わない

**プロセス特徴（NG: ラベルが長い、コメントが曖昧）**
- 段階的にリスクを最小化しながら導入していくアプローチ ← ラベルが長すぎ
  - いい感じに進める ← 曖昧
  - 適切に対応する ← 具体性ゼロ

---

## テーマ：新規事業立ち上げプロセス（9ステップ）

### ✅ 良い例

**Main Message**
市場機会を逃さないため、9段階のステージゲート型プロセスで新規事業を迅速に立ち上げるべき

**Chart Title**
新規事業立ち上げプロセス

**プロセスステップ**
1. 機会探索
2. 仮説構築
3. 顧客検証
4. MVP開発
5. 市場テスト
6. 事業計画策定
7. 資金調達
8. 本格展開
9. 事業評価

**プロセス特徴**
- 仮説検証型：顧客インタビューとMVPテストを重ね、事業仮説の精度を高める / 市場テスト段階で撤退基準を明確化し、損切り判断を迅速に行う
- スピード重視：機会探索からMVP開発まで90日以内で完了する目標を設定する / 並行作業が可能なフェーズは同時進行し、全体リードタイムを短縮する
- 経営連携：各ステージゲートで経営会議に報告し、Go/No-Go判断を仰ぐ / 資金調達段階では外部投資家も含めた評価委員会を組成する
