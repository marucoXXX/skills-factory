# 競合比較サマリー PowerPoint ジェネレーター

対象会社＋競合3〜5社の比較サマリースライド（1枚）を生成するスキル。

**方式: PPTXネイティブテーブル**
- 全オブジェクトがPPT上で編集可能（画像化しない）
- python-pptx の `add_table()` で動的にテーブルを構築

---

## スライド構成

| セクション | 位置 | 内容 |
|---|---|---|
| **メインメッセージ** | 最上部 | 最大65文字（hard-fail）。「〜すべき」で締める |
| **チャートタイトル** | メインメッセージ直下 | 「対象会社 対 主要競合N社：競合比較サマリー」等 |
| **比較テーブル** | コンテンツエリア | 横型テーブル（行=比較項目、列=企業） |
| **出典** | 左下 | 情報ソースの一括記載 |

### 比較テーブルの仕様

- **列構成（target_company を指定した場合）**:
  - 左端: 比較項目ラベル列（幅14%、グレー背景）
  - 2列目: 対象会社列（**イエロー背景 #FFF4C2 ＋太字**）
  - 3列目以降: 競合企業列（残り幅を等分、偶数行のみ淡グレー背景）

- **列構成（target_company を指定しない場合・推奨デフォルト）**:
  - 左端: 比較項目ラベル列（幅14%、グレー背景）
  - 2列目以降: `competitors[]` を順に並べる（**強調列なし**、偶数行のみ淡グレー背景）

- **行構成**:
  - ヘッダー行: 企業名（太字、中央寄せ、高さ0.40"）
  - データ行: 比較項目の値（左寄せ、残りの高さを等分）

- **行数**: ヘッダー1行 + データ7行 = 8行（比較項目数は可変だが7項目が標準）

- **列数**: 比較項目列1 + 企業列N（target あり: 1+対象+競合 / target なし: 1+全社）

---

## 入力パターンと処理フロー

### パターンA：生情報（IM・HP・議事録等）が入力された場合

**いきなりPowerPointを作成しない**

1. **Step 1: 各社の情報を抽出・整理する**
2. **Step 2: Markdownでユーザーに提示**し、確認・修正を求める
3. **Step 3: ユーザーの承認後**、PowerPointを生成する

### パターンB：データが整理済み（JSON/表形式）で入力された場合

1. **Step 1: 内容を確認**し、項目が揃っているか確認
2. **Step 2: 確認後、PowerPointを生成する**

### パターンC：他スキル（`competitor-analysis-pptx`）から呼ばれる場合

1. 渡されたJSONデータを使用し、直接PowerPointを生成する

## Step 3: PowerPointの生成

### JSONデータ仕様

`{{WORK_DIR}}/competitor_summary_data.json` に以下の形式で保存する：

```json
{
  "main_message": "対象会社は業界3位のポジションにあり、上位2社との規模ギャップ・差別化戦略を多面的に評価すべき",
  "chart_title": "対象会社 対 主要競合3社：競合比較サマリー",
  "source": "出典：各社HP、IR資料（2025年度）、東京商工リサーチ、日経バリューサーチ",
  "target_company": {
    "name": "株式会社 対象ABC",
    "事業内容": "産業用機械部品の製造・販売",
    "本社所在地": "愛知県名古屋市",
    "設立年": "1975年",
    "従業員数": "850名（連結）",
    "売上高": "125億円(2025年3月期)",
    "上場区分": "未上場",
    "強み・差別化": "高精度加工技術、短納期対応力"
  },
  "competitors": [
    {
      "name": "競合X工業",
      "事業内容": "...",
      "本社所在地": "...",
      "設立年": "...",
      "従業員数": "...",
      "売上高": "...",
      "上場区分": "...",
      "強み・差別化": "..."
    }
  ],
  "comparison_items": [
    {"key": "事業内容", "label": "事業内容"},
    {"key": "本社所在地", "label": "本社所在地"},
    {"key": "設立年", "label": "設立年"},
    {"key": "従業員数", "label": "従業員数"},
    {"key": "売上高", "label": "売上高"},
    {"key": "上場区分", "label": "上場区分"},
    {"key": "強み・差別化", "label": "強み・差別化"}
  ]
}
```

### JSONフィールド仕様

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `main_message` | string | 必須 | メインメッセージ。最大65文字（hard-fail）、「〜すべき」で締める |
| `chart_title` | string | 任意 | チャートタイトル。デフォルト「競合比較」 |
| `source` | string | 任意 | 出典（左下に一括表示） |
| `target_company` | object | 任意 | 強調したい会社の情報。指定するとその会社の列がイエロー背景＋太字でハイライトされる。**未指定なら competitors[] のみで全社フラット表示（強調なし）**。`name` + 各 comparison_item の key と対応する値を持つ |
| `target_company.name` | string | 任意 | 対象会社名（target_company を指定する場合のみ必須） |
| `competitors` | array | 必須 | 競合の配列（3〜5要素推奨） |
| `competitors[].name` | string | 必須 | 競合企業名 |
| `comparison_items` | array | 必須 | 比較項目の定義（行になる）。標準は7項目 |
| `comparison_items[].key` | string | 必須 | データ参照キー（target_company・competitorsの中のキーと一致させる） |
| `comparison_items[].label` | string | 必須 | 左端列に表示されるラベル |

### 売上高の年度ルール

- **実行年の前年**（例：2026年実行なら2025年度）を自動取得する
- 形式例：「125億円（2025年3月期）」「680億円（2025年12月期）」
- 決算期が不明な場合は「125億円（2025年度）」

### 比較項目のカスタマイズ

デフォルトの7項目から変更したい場合：
- `comparison_items` の配列を編集し、`target_company`・`competitors` の各オブジェクトに対応するキーを追加する
- `key` と `label` は一致させる必要はない（例：`key="hq_location"`, `label="本社所在地"` も可能）

## デザイン仕様

### フォントサイズ（動的調整）

`get_font_sizes()` は **総表示企業数（= 競合数 + (target あり時 1) ）** に基づいて決定する。下表は target_company を指定した場合の代表値:

| 競合社数 | 総列数（target+比較項目列含む）| ヘッダー | 本文 | 項目ラベル |
|---|---|---|---|---|
| 3社 | 5列 | 14pt Bold | 13pt | 14pt Bold |
| 4社 | 6列 | 13pt Bold | 12pt | 13pt Bold |
| 5社 | 7列 | 12pt Bold | 11pt | 12pt Bold |

`target_company` 未指定時は総表示企業数が 1 少なくなるため、相対的に 1 段階大きいフォントが適用される（同じ閾値表に従う）。

> ⚠️ **2026-04-29 変更**: v0.2 Phase A で competitors の上限を 5→10 に拡張したが、9 列構成（target+8）で cell 30字制限の運用負荷が高いため **5 社上限へ撤回**（ISSUE-008）。本表も 3〜5 社の 3 段階のみに簡素化。

### フォント

| 要素 | フォント |
|---|---|
| 日本語 | Meiryo UI |
| ラテン文字 | Arial |

### 色

| 要素 | カラーコード |
|---|---|
| テキスト | #333333 |
| 対象会社列背景（ハイライト） | #FFF4C2（イエロー） |
| 比較項目列背景 | #F0F0F0（グレー） |
| 偶数行背景 | #FAFAFA |
| 奇数行背景 | #FFFFFF |

### レイアウト定数

| 要素 | 値 |
|---|---|
| スライドサイズ | 13.33" × 7.50"（ワイドスクリーン） |
| コンテンツ左端 | 0.41" |
| コンテンツ上端 | 1.50" |
| コンテンツ幅 | 12.52" |
| コンテンツ下端 | 6.90"（出典の上） |
| 比較項目列幅 | 総幅の14% |
| ヘッダー行高 | 0.40" |
| データ行高 | 残りの高さ÷項目数（項目7で約0.71"） |

## 品質チェックリスト

- [ ] メインメッセージが正しく表示されているか
- [ ] チャートタイトルが正しく表示されているか

## オーケストレーター連携

`market-overview-agent` から呼び出される場合の規約：

| 項目 | 値 |
|---|---|
| 入力JSONファイル名 | `data_08_competitor_summary.json` |
| 出力PPTXファイル名 | `slide_08_competitor_summary.pptx` |
| 入力ディレクトリ | `{{WORK_DIR}}/<run_id>/`（オーケストレーター作業領域） |
| 出力ディレクトリ | 同上 |
| 競合5社上限 | デッキ全体で同じ5社（market-overview-agent が一貫性を担保）|
| 比較項目の調整 | market-overview-agent では「事業内容 / 主力製品 / 主要顧客 / 売上高 / 戦略の重点 / グローバル展開度 / 強み」等の市場分析特化7項目への置換も可 |

オーケストレーターは `merge_order.json` の `entries[]` に
`{ "slide_number": 8, "skill_name": "competitor-summary-pptx", "data_file": "data_08_competitor_summary.json", "file_name": "slide_08_competitor_summary.pptx" }`
を登録する。

`strategy-report-agent` から呼び出される場合は、上記とは別の番号付け規約を持つ。混同を避けるため、
オーケストレーター側で `data_file` / `file_name` を明示的に指定して呼び出すこと。
