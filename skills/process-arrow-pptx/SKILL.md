# プロセス矢羽 PowerPoint ジェネレーター

戦略コンサルティングで頻繁に使われる**プロセス矢羽（Process Arrow / Chevron）チャート**を生成するスキル。
3ステップまたは5ステップのプロセスを矢羽で視覚化し、各ステップに対応する2段の補足情報を記載する。

---

## プロセス矢羽チャートとは

ビジネスの手順・フェーズ・プロセスを左から右に流れる矢羽（シェブロン/ペンタゴン型図形）で表現するチャート。
各矢羽にステップ名を記載し、矢羽の下に2段のラベル行で補足情報（目的と主要施策、インプットとアウトプット、課題と施策など）を記載する。

### チャートの構成要素

| 要素 | 定義 | ポイント |
|------|------|------|
| **Main Message** | チャート全体の結論・示唆を述べる一文。最大70文字 | 必ず「〜すべき」で締める。読み手にアクションを促す |
| **Chart Title** | チャートのテーマ・文脈を端的に示す短いフレーズ（10〜20文字） | Main Messageを補足する文脈を示す |
| **矢羽 (Arrow)** | 各ステップ/フェーズの名前。矢羽図形の中に表示される | 短く明確に。2〜6文字程度 |
| **Label1** | 上段のラベル名。全矢羽に共通の補足軸（例：「目的」「インプット」「概要」） | 太字で表示 |
| **Label2** | 下段のラベル名。全矢羽に共通の補足軸（例：「主要施策」「アウトプット」「成果物」） | 太字で表示 |
| **Arrow N — Label1** | 各矢羽のLabel1に対応する詳細テキスト | 具体的に。1〜2文で |
| **Arrow N — Label2** | 各矢羽のLabel2に対応する詳細テキスト | 具体的に。1〜2文で |

### Main Messageの記述ルール

Main MessageはSCRスキルと同じ原則に従う。チャート全体を一言で総括し、読み手に行動を促す文にする。

- 最大70文字
- 必ず「〜すべき」で終える
- プロセス全体の意義・目的を端的に伝える
- 具体的な数字や固有名詞を入れるとより説得力が増す

**良い例:**
- 「新規事業立ち上げは5フェーズで推進し、Q3までにPMF検証を完了すべき」
- 「コスト削減プログラムは3段階のアプローチで年間2億円の効果を実現すべき」
- 「DX推進は現状分析→施策設計→実行の3ステップで全社展開すべき」

**悪い例（避けるべき）:**
- 「プロセスの概要」（結論がない、「〜すべき」がない）
- 「以下の手順で進める」（Main Messageとして弱い）
- 「DXについて」（具体性がない）

### Chart Titleの記述ルール

- 10〜20文字程度
- Main Messageの文脈やテーマを補足する
- 例：「新規事業推進プロセス」「コスト最適化の全体像」「DXロードマップ」

## テンプレート選択

| テンプレート | ファイル | 用途 |
|---|---|---|
| 3矢羽 | `assets/ProcessArrow3.pptx` | 大きな3フェーズで表現する場合。各フェーズの詳細を記述するスペースが広い |
| 5矢羽 | `assets/ProcessArrow5.pptx` | より細かい5ステップで表現する場合。各ステップはコンパクトに記述 |

選び方の目安：
- プロセスが3つ以下の大きなフェーズに分けられる場合 → **3矢羽**
- プロセスが4〜5の明確なステップに分けられる場合 → **5矢羽**
- 議事録から抽出する場合、内容の粒度に応じて適切な方を選ぶ

## 入力パターンと処理フロー

### パターンA：議事録・文字起こしが入力された場合

**いきなりPowerPointを作成しない**

1. **Step 1: プロセス矢羽ドラフトを作成する**
   - 議事録の内容を分析し、プロセスやフェーズを特定する
   - 3矢羽か5矢羽かを判断する（ユーザーが指定していなければ提案する）
   - 各矢羽の名前、Label1/Label2の軸名、各矢羽の詳細をドラフトする
   - Main MessageとChart Titleをドラフトする
2. **Step 2: Markdownでユーザーに提示**し、確認・修正を求める
3. **Step 3: ユーザーの承認後**、PowerPointを生成する

### パターンB：ユーザーから詳細な指示がある場合

1. **Step 1: 内容を確認**し、矢羽数・ラベル・詳細が揃っているか確認
2. 不足・曖昧な点があれば補完提案を行う
3. **Step 2: 確認後、PowerPointを生成する**

---

## Step 1: 議事録からプロセスを抽出する

### プロセスの特定

議事録から時系列・順序性のある活動を抽出する：
- 「まず〜、次に〜、最後に〜」のような順序表現
- 「Phase 1」「フェーズ1」「Step 1」のような明示的なフェーズ名
- 「〜した後で」「〜の前に」のような時系列の前後関係
- 「準備→実行→振り返り」のようなプロセスフレームワークへの言及

### 矢羽数の判断基準

- 議事録に3つの大きなフェーズが見える → 3矢羽
- 議事録に4〜5つの明確なステップがある → 5矢羽
- 曖昧な場合は、内容の粒度に応じて3矢羽（粗い粒度）か5矢羽（細かい粒度）を提案

### Label軸の選び方

プロセスの性質に応じて適切なLabel軸を選ぶ：

| プロセスの種類 | Label1 の例 | Label2 の例 |
|---|---|---|
| プロジェクト推進 | 目的 | 主要施策 |
| 業務フロー | インプット | アウトプット |
| 変革プログラム | 概要 | 成果物 |
| 戦略立案 | 論点 | アクション |
| 製品開発 | 活動内容 | マイルストーン |

## Step 3: PowerPointの生成

### データのJSON化

矢羽内容を `{{WORK_DIR}}/arrow_data.json` に以下の形式で保存する：

```json
{
  "main_message": "〜すべき（最大70文字）",
  "chart_title": "〜のテーマ（10〜20文字）",
  "label1": "目的",
  "label2": "主要施策",
  "arrows": [
    {
      "name": "ステップA",
      "label1_detail": "Label1に対応する詳細",
      "label2_detail": "Label2に対応する詳細"
    },
    {
      "name": "ステップB",
      "label1_detail": "Label1に対応する詳細",
      "label2_detail": "Label2に対応する詳細"
    },
    {
      "name": "ステップC",
      "label1_detail": "Label1に対応する詳細",
      "label2_detail": "Label2に対応する詳細"
    }
  ]
}
```

矢羽の数（`arrows`配列の長さ）は必ず3または5にすること。これによって使用するテンプレートが決まる。

### テンプレートの選択

```bash
# 3矢羽の場合
TEMPLATE="<skill-path>/assets/ProcessArrow3.pptx"
# 5矢羽の場合
TEMPLATE="<skill-path>/assets/ProcessArrow5.pptx"
```

`<skill-path>` はこのスキルのインストールパスに置き換える。

### スクリプト実行コマンド

```bash
python <skill-path>/scripts/fill_process_arrow.py \
  --data {{WORK_DIR}}/arrow_data.json \
  --template "$TEMPLATE" \
  --output {{OUTPUT_DIR}}/ProcessArrow_output.pptx
```

### 出力確認

```bash
python -m markitdown {{OUTPUT_DIR}}/ProcessArrow_output.pptx
```

内容が正しく反映されているか確認し、ユーザーに提示する。

## 品質チェックリスト

PowerPoint生成後、以下を確認：

- [ ] Main Messageが70文字以内で「〜すべき」で終わっているか
- [ ] Chart Titleが10〜20文字程度で文脈を的確に示しているか
- [ ] 矢羽の数がテンプレートと一致しているか（3 or 5）
- [ ] 全ての矢羽に名前が入っているか
- [ ] Label1/Label2のラベル名が設定されているか
- [ ] 各矢羽のLabel1/Label2詳細が入っているか
- [ ] PPTXのmarkitdown出力でプレースホルダーテキスト（Arrow1, Label1等）が残っていないか
