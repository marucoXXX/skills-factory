# 市場環境分析 PowerPoint ジェネレーター

BDD（ビジネスデュー・ディリジェンス）や事業分析において、業界の市場規模・推移を
積み上げ棒グラフ＋折れ線グラフ＋CAGR注釈で1枚のPowerPointスライドに整理するスキル。

---

## スライド構成

| セクション | 位置 | 内容 |
|---|---|---|
| **メインメッセージ** | 最上部 | 最大65文字（hard-fail）。「〜すべき」で締める |
| **チャートタイトル** | メインメッセージ直下 | 「○○業の○○の推移（全国）」等 |
| **チャートエリア** | 左側（約72%） | 積み上げ棒グラフ＋折れ線グラフ＋成長率注釈 |
| **CAGR注釈** | 右側（約28%） | 期間別CAGR（系列ごと） |
| **出典** | 左下 | 情報ソースの記載 |

### チャートエリアの構成

- **積み上げ棒グラフ**: 2〜5系列の積み上げ（例: 国内/海外、製品A/B/C）。**左 Y 軸**にスケール
- **折れ線グラフ**: 比率や指標のトレンド（例: 構成比率、利益率）。**右 Y 軸**にスケール（dual-axis）
- **Y 軸の自動レンジ**: 棒の左軸は積み上げ合計 ×1.15 を nice round（1.0, 1.5, 2.0, 2.5, ...）で自動決定。折れ線の右軸は値域が 0-100 なら 120、それ以外は最大値 ×1.15 を自動丸め。`primary_y_axis_max` / `secondary_y_axis_max` で明示指定も可能
- **成長率注釈**: 特定期間のCAGRを矢印＋楕円で表示
- **期間区切り線**: データの不連続（推計方法変更等）を点線で表示
- **各セグメントのデータラベル**: 棒内部に白文字で表示
- **トータルラベル**: 棒の上に合計値を表示
- **凡例**: チャート下部中央

### 右側CAGR注釈の構成

- 期間ごとに区分（例: CAGR 2013→2019、CAGR 2020→2022）
- 各区分内に系列ごとのCAGR値を表示

## 入力パターンと処理フロー

### パターンA：業界名・業種のみが入力された場合（メインフロー）

**いきなりPowerPointを作成しない。必ず以下のステップを踏む。**

#### Step 1: 統計ソースの探索

ユーザーが「○○業の市場環境スライドを作って」のように業界名のみ指定した場合、
まず **web_search** で当該業界の統計ソース候補を複数検索する。

**検索クエリの組み合わせ（3〜5回検索）:**

```
1. 「○○業 市場規模 推移 統計」
2. 「○○業 年次統計 官公庁」
3. 「○○業 業界団体 統計データ」
4. 「○○ 産業 動態統計 経済産業省」
5. 「○○業界 市場規模 レポート」
```

**web_fetch** で上位候補のページ内容を確認し、以下を把握する:
- データの時系列範囲（何年分あるか）
- データの粒度（月次/四半期/年次）
- セグメント分解の有無（内訳があるか）
- データの信頼性（官公庁 > 業界団体 > 調査会社）

#### Step 2: ソース候補の提示と議論

見つかった統計ソース候補を **3〜5つ** ユーザーに提示し、どれを使うか議論する。

**提示フォーマット:**

```markdown
## 統計ソース候補

「○○業」の市場環境分析に使える統計ソースを調査しました。

### 候補1: ○○省「○○統計調査」 ⭐推奨
- **URL**: https://...
- **データ範囲**: 2010年〜2023年（14年分）
- **更新頻度**: 年次
- **内訳**: セグメント別、地域別あり
- **メリット**: 官公庁統計で信頼性が高い。長期時系列あり
- **デメリット**: 最新年度の公開が1年遅れ

### 候補2: ○○業界団体「○○年報」
- **URL**: https://...
- **データ範囲**: 2018年〜2023年（6年分）
- **更新頻度**: 年次
- **内訳**: 工種別あり
- **メリット**: 業界固有の詳細分類あり
- **デメリット**: 時系列が短い

### 候補3: ○○研究所「○○市場動向レポート」
- **URL**: https://...
- **データ範囲**: 2020年〜2025年予測
- **更新頻度**: 年次
- **内訳**: なし（市場規模のみ）
- **メリット**: 将来予測あり
- **デメリット**: 有料レポートの概要のみ。データの透明性が低い
```

**ask_user_input ツールを使って選択肢を提示する:**

```
質問: どの統計ソースを使用しますか？
選択肢:
  - 候補1: ○○省「○○統計調査」（推奨）
  - 候補2: ○○業界団体「○○年報」
  - 候補3: ○○研究所レポート
  - 複数組み合わせて使いたい
```

#### Step 3: 選択されたソースからデータ収集

ユーザーが選択したソースから **web_fetch** でデータを取得・抽出する。

**データ収集のポイント:**
- **時系列**: 最低5年分、理想は10年分
- **内訳**: 可能であれば2〜3つのセグメントに分解（例: 国内/海外、技術別、用途別）
- **比率指標**: 構成比率やマージン率など折れ線で表現できる指標
- **データの不連続**: 推計方法の変更など、時系列の連続性に影響する情報
- **単位**: 億円、兆円、千人など、スライドで見やすい単位に変換

#### Step 4: データ整理結果の確認

抽出・整理したデータを **Markdown** でユーザーに提示し、確認・修正を求める。

**このステップで以下も合わせて議論する:**
- チャートの系列構成（何を棒に、何を折れ線にするか）
- CAGR計算の期間区分
- 期間区切り線の要否
- メインメッセージのドラフト

#### Step 5: ユーザー承認後、PowerPointを生成

---

### パターンB：特定のURLや統計名が指定された場合

1. **Step 1**: 指定されたURLを **web_fetch** でデータ取得
2. **Step 2**: データ整理結果をMarkdownで提示、確認
3. **Step 3**: 承認後、PowerPoint生成

### パターンC：データが整理済みで入力された場合

1. **Step 1**: 内容を確認し、データの整合性を確認
2. **Step 2**: 確認後、PowerPoint生成

## ソース選択の判断基準

### 優先する情報ソース

| 優先度 | ソース種別 | 例 | 特徴 |
|---|---|---|---|
| 1 | 官公庁統計 | 国土交通省、経済産業省、総務省、厚生労働省 | 長期時系列、高信頼性、無料 |
| 2 | 業界団体統計 | 日本建設業連合会、日本自動車工業会 | 業界固有の詳細データ |

| 医療 | 医療施設動態調査 | 厚生労働省 |

`{{WORK_DIR}}/market_environment_data.json` に以下の形式で保存する：

```json
    "unit_label": "（単位：億円）",
    "num_format": "0",

    "secondary_y_axis_max": null,

      {"series_name": "セグメントB", "color": "#E8923F"},

      {

          {"name": "セグメントA", "value": "+X.X%"},

```

### JSONフィールド仕様

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `main_message` | string | 必須 | メインメッセージ。最大65文字（hard-fail）、「〜すべき」で締める |
| `chart_title` | string | 任意 | チャートタイトル。デフォルト「市場環境分析」 |
| `source` | string | 任意 | 出典テキスト（左下に表示） |
| `chart.unit_label` | string | 任意 | 単位表記（左上に表示） |
| `chart.num_format` | string | 任意 | データラベルの数値書式。デフォルト "0.0"（小数1桁）。整数なら "0"、カンマ区切りなら "#,##0" |
| `chart.primary_y_axis_max` | number | 任意 | 左 Y 軸（棒グラフ用）の上限値を明示指定。未指定の場合は積み上げ合計の最大値 ×1.15 を見栄えのよい数値（1, 1.2, 1.5, 2, 2.5, 3, ... ×10^n）に丸めて自動決定 |
| `chart.secondary_y_axis_max` | number | 任意 | 右 Y 軸（折れ線用）の上限値を明示指定。未指定で折れ線値が 0-100 の範囲なら 120、それ以外は最大値 ×1.15 を自動丸め |
| `chart.stacked_bars` | array | 必須 | 積み上げ棒グラフの系列定義（下から順） |
| `chart.stacked_bars[].series_name` | string | 必須 | 系列名 |
| `chart.stacked_bars[].color` | string | 必須 | 色コード（例: "#4E79A7"） |
| `chart.line` | object | 任意 | 折れ線グラフの系列定義。省略可 |
| `chart.line.series_name` | string | 必須 | 系列名 |
| `chart.line.color` | string | 任意 | 色コード。デフォルト "#333333" |
| `chart.line.marker_size` | int | 任意 | マーカーサイズ。デフォルト 8 |
| `chart.line.num_format` | string | 任意 | 右 Y 軸の数値書式。デフォルト "0"。パーセント表示なら `"0\"%\""`、小数1桁なら "0.0" |
| `chart.data` | array | 必須 | 年度別データの配列 |
| `chart.data[].year` | string | 必須 | 年度ラベル（例: "2023"） |
| `chart.data[].bars` | array | 必須 | 各系列の値（stacked_barsの順序に対応） |
| `chart.data[].line_value` | number | 任意 | 折れ線の値 |
| `chart.data[].total_label` | string | 任意 | 棒の上に表示する合計ラベル。空文字で非表示 |
| `chart.period_separator` | object | 任意 | 期間区切り線 |
| `chart.period_separator.after_index` | int | 必須 | 区切り線の位置（dataの何番目の後か） |
| `chart.period_separator.label` | string | 任意 | 区切り線のラベル |
| `chart.cagr_annotations` | array | 任意 | 右側CAGR注釈の配列 |
| `chart.cagr_annotations[].label` | string | 必須 | 期間ラベル（例: "CAGR（2013→2019）"） |
| `chart.cagr_annotations[].items` | array | 必須 | 系列ごとのCAGR値 |
| `chart.growth_annotations` | array | 任意 | チャート上部の成長率注釈（矢印＋楕円） |
| `chart.growth_annotations[].start_index` | int | 必須 | 矢印の開始位置（dataインデックス） |
| `chart.growth_annotations[].end_index` | int | 必須 | 矢印の終了位置（dataインデックス） |
| `chart.growth_annotations[].value` | string | 必須 | 成長率の表示値（例: "+2.4%"） |

## チャートのバリエーション

### 折れ線なしパターン

`chart.line` を省略すると、積み上げ棒グラフのみのチャートになる。

### 単一棒グラフパターン

`chart.stacked_bars` を1系列にすると、単一色の棒グラフになる。

### 期間区切りなしパターン

`chart.period_separator` を省略すると、区切り線なしの連続チャートになる。

### 成長率注釈なしパターン

`chart.growth_annotations` を省略または空配列にすると、矢印＋楕円の注釈なしになる。

## オーケストレーター連携

### `market-overview-agent` から呼び出される場合の規約

| 項目 | 値 |
|---|---|
| 入力JSONファイル名 | `data_04_market_environment.json` |
| 出力PPTXファイル名 | `slide_04_market_environment.pptx` |
| 入力ディレクトリ | `{{WORK_DIR}}/<run_id>/`（オーケストレーター作業領域） |
| 出力ディレクトリ | 同上 |
| 競合5社上限 | デッキ全体で同じ5社を採用（market-overview-agent が一貫性を担保） |

オーケストレーターは `merge_order.json` の `entries[]` に
`{ "slide_number": 4, "skill_name": "market-environment-pptx", "data_file": "data_04_market_environment.json", "file_name": "slide_04_market_environment.pptx" }`
を登録する。

### `business-deepdive-agent` から呼び出される場合の規約

| 項目 | 値 |
|---|---|
| 入力 JSON ファイル名 | `data_NN_market_environment.json`（NN は global_slide_offset 経由で親が採番） |
| 出力 PPTX ファイル名 | `slide_NN_market_environment.pptx`（同上） |
| 入力ディレクトリ | `{{WORK_DIR}}/company-deepdive-agent/<parent_run_id>/segments/<segment_slug>/` |
| 出力ディレクトリ | 同上 |

`business-deepdive-agent` は本スキルを **5 論点中 5 番目（顧客は成長するか？）** として呼び出す。
ここでの市場は会社全体の事業市場ではなく、**対象セグメントの顧客側市場**（例: タクシー事業なら観光・移動需要市場）を扱う。
作業ディレクトリは `company-deepdive-agent` 配下のセグメント別 subdir に統一し、merge は親が担当。
