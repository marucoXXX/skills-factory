# ガントチャート PowerPoint ジェネレーター

プロジェクトの全体スケジュールをガントチャート形式で可視化し、戦略コンサルタント品質のPowerPointスライドを生成するスキル。
HTML/CSSで高品質なガントチャートを描画し、Playwrightでスクリーンショットを撮影してPPTXに挿入する方式。

---

## ガントチャートの構成要素

| 要素 | 定義 | ポイント |
|------|------|------|
| **Main Message** | スケジュール全体を総括し、読み手にアクションを促す一文。最大70文字 | 必ず「〜すべき」で締める |
| **Chart Title** | スライドのサブタイトル。スケジュールのテーマや文脈を端的に示す短いフレーズ（10〜20文字） | Main Messageを補足する文脈を示す |
| **フェーズ** | タスクをグルーピングする大分類（色付きヘッダーバー） | Phase 1, Phase 2, ... のように構造化 |
| **タスクバー** | 各タスクの期間を横棒で表示。進捗率を濃淡で表現 | 開始日・終了日・担当・進捗率 |
| **マイルストーン** | 重要な節目を◆マークと垂直線で表示 | 「要件FIX」「Go-Live」など |
| **月ヘッダー** | タイムラインの月区切りをヘッダーに表示 | 自動計算 |

### 記述ルール

- **Main Message**: 最大70文字。スケジュール全体を一言で総括し、必ず「〜すべき」で終える。
- **Chart Title**: 10〜20文字程度。Main Messageのテーマを補足する文脈フレーズ。
- **フェーズ**: プロジェクトの大きな区切り。色で視覚的に区別される。
- **タスク**: 各タスクに開始日・終了日は必須。担当者・進捗率は任意。
- **日付形式**: `YYYY-MM-DD` または `YYYY/MM/DD`。
- **ページ分割**: 1スライドあたり最大15行（フェーズヘッダー含む）。超過時は自動分割。

## 入力パターンと処理フロー

### パターンA：議事録・文字起こしが入力された場合

**いきなりPowerPointを作成しない**

1. **Step 1: スケジュールのドラフトを作成する**（後述の抽出ガイドラインに従う）
2. **Step 2: Markdownでユーザーに提示**し、確認・修正を求める
3. **Step 3: ユーザーの承認後**、PowerPointを生成する

### パターンB：スケジュールデータが明確に入力された場合

1. **Step 1: 内容を確認**し、フェーズ・タスク・日付が揃っているか確認
2. 不足・曖昧な点があれば修正提案を行う
3. **Step 2: 確認後、PowerPointを生成する**

---

## Step 1: 議事録からスケジュールを抽出する

### フェーズの特定

議事録から**プロジェクトの大きな区切り**を拾う：
- 「Phase 1」「第1フェーズ」「要件定義フェーズ」
- 「〜の段階」「〜のステップ」
- 明確なフェーズ分けがなければ、作業内容から論理的にグルーピングする

### タスクの抽出

各フェーズ内の**具体的な作業項目**を拾う：
- 「〜の実施」「〜の作成」「〜のレビュー」
- 開始日・終了日・期間の言及を探す
- 担当者の言及があれば紐づける

### マイルストーンの特定

プロジェクトの**重要な節目**を拾う：
- 「〜完了」「〜承認」「Go/No-Go判定」「カットオーバー」
- 「〜までに」「デッドライン」「期限」

- 要件FIX: 2025/05/15

## Step 3: PowerPointの生成

### テンプレートの参照

テンプレートは `assets/gantt-chart-template.pptx` を使用する。

```bash
TEMPLATE="<SKILL_DIR>/assets/gantt-chart-template.pptx"
```

### スケジュールデータのJSON化

内容を `{{WORK_DIR}}/gantt_data.json` に以下の形式で保存する：

```json
{
  "main_message": "〜すべき（最大70文字）",
  "chart_title": "〜スケジュール（10〜20文字）",
  "schedule_start": "2025-04-01",
  "schedule_end": "2025-12-31",
  "phases": [
    {
      "name": "Phase 1: 要件定義",
      "color": "#1565C0",
      "tasks": [
        {
          "name": "タスク名",
          "start": "2025-04-01",
          "end": "2025-04-30",
          "owner": "担当者",
          "progress": 100
        }
      ]
    }
  ],
  "milestones": [
    {"name": "要件FIX", "date": "2025-05-15"}
  ]
}
```

**フィールド仕様:**

| フィールド | 必須 | 説明 |
|---|---|---|
| `schedule_start` | △ | タイムラインの開始日。未指定時はタスクの最早開始日から自動計算 |
| `schedule_end` | △ | タイムラインの終了日。未指定時はタスクの最遅終了日から自動計算 |
| `phases[].name` | ◎ | フェーズ名。色付きヘッダーバーとして表示 |
| `phases[].color` | ○ | フェーズの色（16進数）。未指定時はデフォルト色を順番に適用 |
| `phases[].tasks[].name` | ◎ | タスク名 |
| `phases[].tasks[].start` | ◎ | 開始日（YYYY-MM-DD） |
| `phases[].tasks[].end` | ◎ | 終了日（YYYY-MM-DD） |
| `phases[].tasks[].owner` | ○ | 担当者名。バー上に表示 |
| `phases[].tasks[].progress` | ○ | 進捗率（0〜100）。バーの濃淡で表現 |
| `milestones[].name` | ◎ | マイルストーン名 |
| `milestones[].date` | ◎ | マイルストーン日付（YYYY-MM-DD） |

**フェーズなしのフラット構成も可能:**
`phases` の代わりに `tasks` 配列を直接指定すると、フェーズヘッダーなしのフラットなガントチャートになる。

### スクリプト実行コマンド

```bash
apt-get install -y -qq fonts-noto-cjk 2>/dev/null
pip install python-pptx playwright -q --break-system-packages
playwright install chromium 2>/dev/null

python <SKILL_DIR>/scripts/fill_gantt_chart.py \
  --data {{WORK_DIR}}/gantt_data.json \
  --template <SKILL_DIR>/assets/gantt-chart-template.pptx \
  --output {{OUTPUT_DIR}}/GanttChart_output.pptx
```

### 出力確認

```bash
python -m markitdown {{OUTPUT_DIR}}/GanttChart_output.pptx
```

## 品質チェックリスト

- [ ] Main Messageが70文字以内で「〜すべき」で終わっているか
- [ ] Chart Titleが10〜20文字程度で文脈を的確に示しているか
- [ ] 全フェーズ・全タスクがガントチャートに反映されているか
- [ ] タスクバーの期間（開始〜終了）が正しいか
- [ ] 進捗率がバーの濃淡で正しく表現されているか
- [ ] マイルストーンが◆マークで正しい日付に表示されているか
- [ ] 複数ページの場合、Chart Titleにページ番号が正しく付与されているか
