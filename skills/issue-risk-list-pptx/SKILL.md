# 課題・リスク一覧 PowerPoint ジェネレーター

プロジェクトの課題やリスクを一覧表形式で整理し、戦略コンサルタント品質のPowerPointスライドを生成するスキル。
列の項目（カテゴリ・概要・詳細・担当・期日など）はプロジェクトに応じて動的に変更できる。

---

## 課題・リスク一覧とは

プロジェクトで認識されている課題（Issue）やリスク（Risk）を一覧表形式で整理したスライド。Main Messageでアクションを促し、Chart Titleで文脈を示し、テーブルで各課題・リスクの詳細を構造化する。

| 要素 | 定義 | ポイント |
|------|------|------|
| **Main Message** | 一覧全体を総括し、読み手にアクションを促す一文。最大70文字 | 必ず「〜すべき」で締める |
| **Chart Title** | スライドのサブタイトル。一覧のテーマや時点を端的に示す短いフレーズ（10〜20文字） | Main Messageを補足する文脈を示す |
| **列定義（columns）** | テーブルのヘッダー列。プロジェクトに応じて自由に定義可能 | 典型例: カテゴリ、概要、詳細、担当、対応期日 |
| **データ行（rows）** | 各課題・リスクの情報。1行1課題/リスク | 推奨上限8行（スライドの可読性を考慮） |

### 記述ルール

- **Main Message**: 最大70文字。一覧全体を一言で総括し、必ず「〜すべき」で終える。ユーザーが指定した場合はそのまま使用、指定がない場合はドラフトして確認を取る。
- **Chart Title**: 10〜20文字程度。Main Messageのテーマを補足する文脈フレーズ。ユーザーが指定しない場合はドラフトして確認を取る。
- **列定義**: プロジェクトの文脈に応じて柔軟に設定する。ユーザーが指定しない場合は以下のデフォルト5列を使用する：
  - カテゴリ（width_ratio: 1.2）
  - 概要（width_ratio: 1.5）
  - 詳細（width_ratio: 3.5）
  - 担当（width_ratio: 1.0）
  - 対応期日（width_ratio: 1.5）
- **データ行**: 行数に制限なし。1スライドあたり最大4行（16ptフォント前提）で、5行以上は自動的に複数スライドに**バランス分割**される（例: 7行 → 4+3 / 10行 → 4+3+3）。末尾ページに1行だけ残るような不均衡を避ける。複数ページ時はChart Titleに「（1/3）」等のページ番号が自動付与される。
- **フォントサイズ**: デフォルトは**16pt（ヘッダー・データ共通）**。JSON側で `font_size_header` / `font_size_data` にて個別に上書き可能（単位: 百分の一pt、16pt = 1600）。

### 列のカスタマイズ例

プロジェクトによって以下のような列構成が考えられる：

- **標準型**: カテゴリ / 概要 / 詳細 / 担当 / 対応期日
- **ステータス付き型**: カテゴリ / 概要 / 詳細 / ステータス / 担当 / 対応期日
- **優先度付き型**: 優先度 / カテゴリ / 概要 / 対応策 / 担当 / 期日
- **リスク評価型**: リスク項目 / 影響度 / 発生確率 / 対応策 / 担当
- **シンプル型**: No. / 課題・リスク / 対応方針 / 担当 / 期日

## 入力パターンと処理フロー

### パターンA：議事録・文字起こしが入力された場合

**いきなりPowerPointを作成しない**

1. **Step 1: 課題・リスク一覧のドラフトを作成する**（後述の抽出ガイドラインに従う）
2. **Step 2: Markdownでユーザーに提示**し、確認・修正を求める
3. **Step 3: ユーザーの承認後**、PowerPointを生成する

### パターンB：課題・リスクが明確に入力された場合

1. **Step 1: 内容を確認**し、列定義とデータ行が揃っているか確認
2. 不足・曖昧な点があれば修正提案を行う
3. **Step 2: 確認後、PowerPointを生成する**

---

## Step 1: 議事録から課題・リスクを抽出する

### 課題（Issue）の抽出

議事録から**既に顕在化している問題**を拾う。以下のような表現を探す：
- 「〜が問題になっている」「〜が遅延している」「〜ができていない」
- 「課題として〜」「対応が必要な〜」「〜の対策を検討中」
- 進捗報告で赤信号・警告が出ている項目

### リスク（Risk）の抽出

議事録から**将来起こりうる問題**を拾う：
- 「〜のリスクがある」「〜の可能性がある」「〜が懸念される」
- 「もし〜なら」「〜が遅れると〜に影響」
- 前提条件・依存関係に関する不確実性

### 列の決定

議事録の内容に応じて適切な列構成を決定する：
- 担当者の言及があれば「担当」列を含める
- 期日の言及があれば「対応期日」列を含める
- 優先度やステータスの議論があれば該当列を追加する
- ユーザーが特定の列を指定していればそれに従う

### データの整理

- 各課題・リスクを1行にまとめる
- 「詳細」列は具体的なファクトと数字を含める
- 議事録に明示されていない情報（担当・期日等）はドラフトとして「※」を付ける

## Step 2: 課題・リスク一覧のMarkdown出力フォーマット

ユーザーに確認を求める際は、以下のフォーマットで出力する：

```markdown
## 課題・リスク一覧 整理結果

**Main Message（※ドラフト）**
〜すべき（最大70文字）

**Chart Title（※ドラフト）**
〜の一覧（10〜20文字）

### 列定義
カテゴリ / 概要 / 詳細 / 担当 / 対応期日

### 課題・リスク一覧

| カテゴリ | 概要 | 詳細 | 担当 | 対応期日 |
|---|---|---|---|---|
| XXX | YYY | ZZZ | AAA | YYYY/MM/DD |
| ... | ... | ... | ... | ... |
```

確認メッセージ例：
> 上記の課題・リスク一覧でよろしいでしょうか？Main Message、Chart Title、列定義も含めて修正があればお知らせください。確認後にPowerPointを生成します。

## Step 3: PowerPointの生成

### テンプレートの参照

テンプレートは `assets/issue-risk-template.pptx` を使用する。

```bash
TEMPLATE="<SKILL_DIR>/assets/issue-risk-template.pptx"
```

`<SKILL_DIR>` はこのスキルがインストールされたディレクトリパスに置き換えること。

テンプレートを初めて使う前に、以下で構造を確認すること：

```bash
pip install "markitdown[pptx]" python-pptx -q --break-system-packages
python -m markitdown <SKILL_DIR>/assets/issue-risk-template.pptx
```

### 課題・リスクデータのJSON化

内容を `{{WORK_DIR}}/issue_risk_data.json` に以下の形式で保存する：

```json
{
  "main_message": "〜すべき（最大70文字）",
  "chart_title": "〜の一覧（10〜20文字）",
  "columns": [
    {"name": "カテゴリ", "width_ratio": 1.2},
    {"name": "概要", "width_ratio": 1.5},
    {"name": "詳細", "width_ratio": 3.5},
    {"name": "担当", "width_ratio": 1.0},
    {"name": "対応期日", "width_ratio": 1.5}
  ],
  "rows": [
    ["カテゴリ1", "概要1", "詳細1", "担当1", "期日1"],
    ["カテゴリ2", "概要2", "詳細2", "担当2", "期日2"]
  ]
}
```

**列定義の仕様:**
- `name`: 列ヘッダーの表示名
- `width_ratio`: 列幅の相対比率（全列の比率合計に対する割合で自動計算）
- 列数に制限なし（推奨上限6列、スライドの可読性を考慮）

**データ行の仕様:**
- 各行は列定義と同じ順序の文字列配列
- 行数に制限なし（1スライドあたり最大6行で自動ページ分割）
- 複数ページ時はChart Titleに「（1/N）」形式のページ番号が自動付与される

### スクリプト実行コマンド

```bash
python <SKILL_DIR>/scripts/fill_issue_risk.py \
  --data {{WORK_DIR}}/issue_risk_data.json \
  --template <SKILL_DIR>/assets/issue-risk-template.pptx \
  --output {{OUTPUT_DIR}}/IssueRisk_output.pptx
```

### 出力確認

```bash
python -m markitdown {{OUTPUT_DIR}}/IssueRisk_output.pptx
```

内容が正しく反映されているか確認し、ユーザーに提示する。

## 品質チェックリスト

PowerPoint生成後、以下を確認：

- [ ] Main Messageが70文字以内で「〜すべき」で終わっているか
- [ ] Chart Titleが10〜20文字程度で文脈を的確に示しているか
- [ ] 列ヘッダーがすべて正しく反映されているか
- [ ] 全データ行が正しく反映されているか（行数・列数の一致）
- [ ] 複数ページの場合、Chart Titleにページ番号が正しく付与されているか
- [ ] 詳細列に具体的なファクト・数字が含まれているか
- [ ] PPTXのmarkitdown出力でプレースホルダーが残っていないか
