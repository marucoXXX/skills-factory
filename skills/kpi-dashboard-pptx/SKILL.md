# KPI・メトリクスダッシュボード PowerPoint ジェネレーター

KPIやメトリクスをカード形式のダッシュボードで可視化し、戦略コンサルタント品質のPowerPointスライドを生成するスキル。

---

## KPIダッシュボードとは

各KPIを「カード」として3列×2行のグリッドに配置し、値・目標・ステータス・トレンドを一覧で把握できるスライドフォーマット。

| 要素 | 定義 | ポイント |
|------|------|------|
| **Main Message** | ダッシュボード全体を総括し、読み手にアクションを促す一文。最大70文字 | 必ず「〜すべき」で締める |
| **Chart Title** | スライドのサブタイトル。ダッシュボードのテーマや時点を端的に示す短いフレーズ（10〜20文字） | Main Messageを補足する文脈を示す |
| **KPIカード** | 各KPIの詳細を表示するカード。name/value/unit/target/status/trend で構成 | 1スライドあたり最大6カード（3列×2行） |

### 記述ルール

- **Main Message**: 最大70文字。ダッシュボード全体を一言で総括し、必ず「〜すべき」で終える。ユーザーが指定した場合はそのまま使用、指定がない場合はドラフトして確認を取る。
- **Chart Title**: 10〜20文字程度。Main Messageのテーマを補足する文脈フレーズ。ユーザーが指定しない場合はドラフトして確認を取る。
- **KPIカード数**: 1スライドあたり最大6カード（3列×2行）。7カード以上は自動ページ分割。
- **ステータス**: 「達成」「注意」「未達」の3段階を推奨。色は自動マッピング（緑/黄/赤）。
- **トレンド**: 「↑」で始まると緑、「↓」で始まると赤、それ以外は黒で表示。
- **ヘッダー色**: テンプレートのAccent2カラーを自動適用。スライドマスターのテーマカラーを変更すれば全カードの色が連動する。

### KPIカードの要素

| フィールド | 必須 | 説明 | 例 |
|---|---|---|---|
| `name` | ◎ | KPI名。ヘッダーバー（Accent2色）に白文字で直接表示 | 「月間売上高」 |
| `value` | ◎ | メトリクスの現在値。24ptの太字で表示 | 「¥4.8億」「18.5%」「142社」 |
| `unit` | ○ | 単位・説明テキスト（14pt、黒） | 「Monthly Recurring Revenue」 |
| `target` | ○ | 目標値（14pt、黒）。「目標: 」が自動付与される | 「¥4.5億」 |
| `status` | ○ | 達成状況（14pt、色付き太字） | 「達成」「注意」「未達」 |
| `trend` | ○ | トレンド情報（14pt、色付き） | 「↑ 前月比 +8.2%」 |

## 入力パターンと処理フロー

### パターンA：議事録・文字起こしが入力された場合

**いきなりPowerPointを作成しない**

1. **Step 1: KPIダッシュボードのドラフトを作成する**（後述の抽出ガイドラインに従う）
2. **Step 2: Markdownでユーザーに提示**し、確認・修正を求める
3. **Step 3: ユーザーの承認後**、PowerPointを生成する

### パターンB：KPIデータが明確に入力された場合

1. **Step 1: 内容を確認**し、各KPIのname/valueが揃っているか確認
2. 不足・曖昧な点があれば修正提案を行う
3. **Step 2: 確認後、PowerPointを生成する**

---

## Step 1: 議事録からKPIを抽出する

### KPIの抽出

議事録から**定量的な指標・メトリクス**を拾う。以下のような表現を探す：
- 数字・割合・金額・件数・スコア
- 「目標は〜」「実績は〜」「KPIは〜」
- 「前月比〜」「前年比〜」「対計画〜」
- 「達成/未達」「オントラック/オフトラック」

### ステータスの判定

- **達成**: 目標値をクリアしている場合
- **注意**: 目標に近いが未達、または悪化傾向の場合
- **未達**: 目標から大幅に乖離している場合

### ヘッダー色について

カードのヘッダーバーにはテンプレートのAccent2カラーが自動適用される。色を変更したい場合は、PowerPointのスライドマスター > テーマカラー > Accent2を編集すれば全カードに反映される。

## Step 2: KPIダッシュボードのMarkdown出力フォーマット

ユーザーに確認を求める際は、以下のフォーマットで出力する：

```markdown
## KPIダッシュボード整理結果

**Main Message（※ドラフト）**
〜すべき（最大70文字）

**Chart Title（※ドラフト）**
〜ダッシュボード（10〜20文字）

### KPI一覧

| No. | KPI名 | 値 | 単位 | 目標 | ステータス | トレンド |
|---|---|---|---|---|---|---|
| 1 | 月間売上高 | ¥4.8億 | MRR | ¥4.5億 | 達成 | ↑ +8.2% |
| 2 | 営業利益率 | 18.5% | OPM | 20.0% | 注意 | ↓ -1.3pt |
```

確認メッセージ例：
> 上記のKPIダッシュボード整理でよろしいでしょうか？Main MessageとChart Titleも含めて修正があればお知らせください。確認後にPowerPointを生成します。

## Step 3: PowerPointの生成

### テンプレートの参照

テンプレートは `assets/kpi-dashboard-template.pptx` を使用する。

```bash
TEMPLATE="<SKILL_DIR>/assets/kpi-dashboard-template.pptx"
```

`<SKILL_DIR>` はこのスキルがインストールされたディレクトリパスに置き換えること。

### KPIデータのJSON化

内容を `/home/claude/kpi_data.json` に以下の形式で保存する：

```json
{
  "main_message": "〜すべき（最大70文字）",
  "chart_title": "〜ダッシュボード（10〜20文字）",
  "kpis": [
    {
      "name": "KPI名",
      "value": "現在値",
      "unit": "単位・説明",
      "target": "目標値",
      "status": "達成|注意|未達",
      "trend": "↑ 前月比 +X%"
    }
  ]
}
```

**KPIの仕様:**
- `name`（必須）: ヘッダーバー（Accent2色）に白文字で直接表示されるKPI名
- `value`（必須）: 24ptの太字で表示されるメトリクス値
- `unit`（任意）: 値の単位・説明テキスト（14pt、黒）
- `target`（任意）: 目標値（14pt、黒）。「目標: 」が自動付与される
- `status`（任意）: 「達成」→緑、「注意」→黄、「未達」→赤で色分け表示（14pt太字）
- `trend`（任意）: トレンド情報（14pt）。「↑」→緑、「↓」→赤で色分け

**ページ分割:**
- 1スライドあたり最大6カード（3列×2行）
- 7カード以上は自動的に複数スライドに分割
- 複数ページ時はChart Titleに「（1/N）」形式のページ番号が自動付与

### スクリプト実行コマンド

```bash
pip install python-pptx -q --break-system-packages
python <SKILL_DIR>/scripts/fill_kpi_dashboard.py \
  --data /home/claude/kpi_data.json \
  --template <SKILL_DIR>/assets/kpi-dashboard-template.pptx \
  --output /mnt/user-data/outputs/KPIDashboard_output.pptx
```

### 出力確認

```bash
python -m markitdown /mnt/user-data/outputs/KPIDashboard_output.pptx
```

内容が正しく反映されているか確認し、ユーザーに提示する。

## 品質チェックリスト

PowerPoint生成後、以下を確認：

- [ ] Main Messageが70文字以内で「〜すべき」で終わっているか
- [ ] Chart Titleが10〜20文字程度で文脈を的確に示しているか
- [ ] 各KPIカードのname/valueが正しく反映されているか
- [ ] ステータスの色分け（達成:緑、注意:黄、未達:赤）が正しいか
- [ ] トレンドの色分け（↑:緑、↓:赤）が正しいか
- [ ] 複数ページの場合、Chart Titleにページ番号が正しく付与されているか
