# PEST分析 PowerPoint ジェネレーター

マクロ環境（外部環境）の4要因を2×2マトリクスで整理し、各項目の影響度（追い風/逆風）も可視化するスキル。
BDD（ビジネスデュー・ディリジェンス）・市場参入検討・中期経営計画策定・投資判断において、
「業界を取り巻くマクロ要因は何か」「何が追い風で何が逆風か」を構造化して示す。

---

## スライド構成

| セクション | 位置 | 内容 |
|---|---|---|
| **メインメッセージ** | 最上部 | 最大70文字。マクロ環境からの示唆を「〜すべき」で締める |
| **チャートタイトル** | メインメッセージ直下 | 「PEST分析：○○市場」 |
| **P: 政治要因 (Political)** | 左上（紫） | 政治・規制・政策の項目 |
| **E: 経済要因 (Economic)** | 右上（青） | 経済・景気・為替・金利の項目 |
| **S: 社会要因 (Social)** | 左下（緑） | 人口動態・価値観・ライフスタイルの項目 |
| **T: 技術要因 (Technological)** | 右下（オレンジ） | 技術革新・R&D・デジタル化の項目 |
| **影響度凡例** | 右下（条件付） | ▲追い風 / ▬中立 / ▼逆風 の凡例（impact指定時のみ） |
| **出典** | 左下 | 情報ソースの記載 |

### PEST配置の論理

```
[P: Political]    [E: Economic]
[S: Social]       [T: Technological]
```

- アクロニム順（P-E-S-T）で左→右、上→下に配置
- 上段はマクロ制度的要因（政策・経済）、下段は需要側要因（社会・技術）のイメージ

### 各象限の構成

- **ヘッダーバー** (高さ0.55in、象限色):
  - **大きな英字** (20pt Bold 白、Arial): "P" / "E" / "S" / "T"
  - **日本語ラベル** (16pt Bold 白、Meiryo UI): "政治要因" / "経済要因" / "社会要因" / "技術要因"
  - **英語ラベル** (11pt 白、Arial、括弧書き): "(Political)" / "(Economic)" / ...
- **ボディ** (薄い象限色、12pt):
  - ブレット記号（•）は象限色
  - 各項目に**影響度インジケーター**（オプション）:
    - **▲ (緑 #1B7A3B)**: 追い風 / Positive
    - **▬ (グレー #666666)**: 中立 / Neutral
    - **▼ (赤 #C03A3A)**: 逆風 / Negative
  - 続いて本文テキスト

### 影響度凡例

JSONデータ内で1つでも `impact` フィールドが指定されていれば、右下に凡例を自動表示。

## 入力パターンと処理フロー

### パターンA：情報が未整理で入力された場合

1. **Step 1: PEST 4象限への分類**（政治・経済・社会・技術）
2. **Step 2: 各項目に影響度を推定**（追い風/中立/逆風）
3. **Step 3: Markdownでユーザーに提示**し、確認・修正を求める
4. **Step 4: ユーザーの承認後**、PowerPointを生成する

### パターンB：PESTが整理済みで入力された場合

1. **Step 1: 内容を確認**し、4象限が揃っているか確認
2. **Step 2: 確認後、PowerPointを生成する**

### PEST 各象限の視点ガイド

| 象限 | 具体項目例 | 情報源の例 |
|---|---|---|
| **P (政治)** | 規制・税制・補助金・業界団体方針・国際関係・通商政策 | 経済産業省、業界団体、金融庁、内閣官房 |
| **E (経済)** | GDP・インフレ・金利・為替・原材料価格・労働市場 | 日銀、内閣府、IMF、シンクタンク |
| **S (社会)** | 人口動態・ライフスタイル・価値観・ESG・教育・健康 | 総務省統計局、博報堂生活総研、シンクタンク |
| **T (技術)** | 技術革新・デジタル化・R&D・特許・インフラ | 経産省、NEDO、業界レポート、学会 |

**項目数の目安**: 各象限 3〜6個（5個前後がバランス良い）

## JSONデータ仕様

`{{WORK_DIR}}/pest_data.json` に以下の形式で保存する：

```json
{
  "source": "出典：経済産業省、METI、IEA、業界レポート（2024年時点）",
  "main_message": "EV市場は政策・社会の追い風と技術進化の恩恵を受ける一方、経済環境と一部規制がリスクとなっており、政策インセンティブを最大活用すべき",
  "chart_title": "PEST分析：日本のEV（電気自動車）市場",
  "pest": {
    "political": {
      "items": [
        {"text": "2035年ガソリン新車販売禁止方針", "impact": "positive"},
        {"text": "CEV補助金・税制優遇の継続", "impact": "positive"},
        {"text": "中国・欧州の自国優遇政策による輸出障壁", "impact": "negative"},
        {"text": "バッテリーリサイクル規制の強化", "impact": "neutral"}
      ]
    },
    "economic": {
      "items": [
        {"text": "円安による原材料調達コスト上昇", "impact": "negative"},
        {"text": "ガソリン価格高止まりによるEV経済性優位", "impact": "positive"}
      ]
    },
    "social": {
      "items": [
        "環境意識の高まり（Z世代・ミレニアル世代）",
        "カーシェア普及による所有離れ"
      ]
    },
    "technological": {
      "items": [
        {"text": "全固体電池の実用化見込み（2027-2030年）", "impact": "positive"},
        {"text": "バッテリーコスト低下（年率7-10%）", "impact": "positive"}
      ]
    }
  }
}
```

### JSONフィールド仕様

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `source` | string | 任意 | 出典テキスト |
| `main_message` | string | 必須 | メインメッセージ（最大70文字） |
| `chart_title` | string | 任意 | チャートタイトル。デフォルト「PEST分析」 |
| `pest.political.items` | array | 必須 | P象限の項目配列 |
| `pest.economic.items` | array | 必須 | E象限の項目配列 |
| `pest.social.items` | array | 必須 | S象限の項目配列 |
| `pest.technological.items` | array | 必須 | T象限の項目配列 |

### 項目 (items) の形式

**Simple形式**（文字列のみ、影響度なし）:
```json
"items": ["項目1", "項目2", "項目3"]
```

**Extended形式**（影響度付き）:
```json
"items": [
  {"text": "項目1", "impact": "positive"},
  {"text": "項目2", "impact": "negative"},
  {"text": "項目3", "impact": "neutral"}
]
```

**両形式を混在させることも可能**:
```json
"items": [
  {"text": "項目1", "impact": "positive"},
  "項目2（影響度なし）",
  {"text": "項目3", "impact": "negative"}
]
```

### `impact` フィールドの受け付ける値

| 意味 | 受付値 | 表示 | 色 |
|---|---|---|---|
| 追い風 | `"positive"`, `"+"`, `"up"`, `"追い風"`, `"pos"` | ▲ | 緑 #1B7A3B |
| 中立 | `"neutral"`, `"="`, `"flat"`, `"中立"`, `"neu"` | ▬ | グレー #666666 |
| 逆風 | `"negative"`, `"-"`, `"down"`, `"逆風"`, `"neg"` | ▼ | 赤 #C03A3A |

### ラベルのカスタマイズ

```json
"political": {
  "label_jp": "政治・規制動向",
  "label_en": "Political & Regulatory",
  "items": [...]
}
```

`label_jp` / `label_en` でヘッダーのラベルを上書き可能（英字 P/E/S/T は固定）。

## 注意事項

- **項目の長さ**: 各項目は40〜50文字以内を推奨。長すぎると折り返しで読みにくくなる
- **影響度の客観性**: 影響度の判定は事業ドメイン（対象会社の業界ポジション）に依存。分析対象を明確にした上で判定する
- **客観性**: 情報源は政府統計・シンクタンク・業界一次データを優先。推測の場合はそれを明示
- **全項目に影響度を付けるか否か**: 混在可能だが、一貫性のため「全て付ける」or「一切付けない」のどちらかを推奨
- **PESTEL拡張**: 環境(Environmental)・法規制(Legal)を追加したPESTELもある。本スキルは標準PEST（4象限）対応。必要ならlegal/environmental項目をP（政治）に統合する

## 競合分析デッキでの使用例

`competitor-analyst-agent` オーケストレータースキルから呼び出される位置づけ:

```
1. 対象会社プロファイル (customer-profile-pptx)
2. 対象会社 事業ポートフォリオ (business-portfolio-pptx)
3. 対象会社 SWOT (swot-pptx)
4. マクロ環境 - PEST分析 ← 本スキル
5. 市場環境 - 市場規模推移 (market-environment-pptx)
6. 市場環境 - 市場シェア分析 (market-share-pptx)
7. 市場環境 - ポジショニングマップ (positioning-map-pptx)
8. 競合比較サマリー (competitor-summary-pptx)
9. 各競合プロファイル (customer-profile-pptx × N)
```

PEST（マクロ環境）→ 市場環境（市場規模・シェア・ポジション）→ 競合分析 の順に
外部環境から具体的な競合までズームインする流れが効果的。

### 他スキルとの使い分け

| スキル | 用途 |
|---|---|
| **PEST分析** | マクロ環境（業界外部の影響要因） |
| SWOT分析 | 対象会社の内外ミックス分析 |
| Five Forces | 業界構造の5つの競争要因 |
| market-environment-pptx | 市場規模の定量推移 |
| positioning-map-pptx | 競合の戦略ポジション |

PESTは**業界全体のマクロ要因**、SWOTは**対象会社固有**、Five Forcesは**業界内の競争圧力** と役割分担する。
