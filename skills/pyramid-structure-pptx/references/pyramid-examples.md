# ピラミッド構造整理の良い例・悪い例

## ピラミッド構造の論理ルール

ピラミッドの各段は**階層的な論理関係**を持つ。並列な論点の羅列ではない。

- **Pyramid1（最上段）**: Main Messageの「So What?（だから何？）」に最も直接的に答える、最重要の根拠
- **Pyramid2（中段）**: Pyramid1を成立させる条件や、行動の緊急性を裏付ける根拠
- **Pyramid3（最下段）**: 実現可能性や基盤的事実を示す、最も具体的な根拠

上位の段は「なぜそう言えるのか？」という問いに対し、下位の段が答える構造になる。

---

## テーマ：海外EC事業の立上げ判断（3段ピラミッド）

### ✅ 良い例

**Main Message**
当社は海外ECプラットフォーム事業を来期中に立ち上げるべき

**Chart Title**
海外EC事業の立上げ判断

**Pyramid 1（最上段）: 収益インパクトの大きさ**
← Main Messageの核心に直結する最重要論拠
- 東南アジアEC市場は年率25%成長で、参入3年目に売上100億円・営業利益率15%が見込める
- 既存国内EC事業の成長率が年率3%に鈍化しており、新たな成長ドライバーが不可欠
- 海外事業の立上げにより、売上構成比の海外比率を現行10%から30%へ引き上げられる

**Pyramid 2（中段）: 参入タイミングの優位性**
← 「なぜ今やるべきか」を裏付け、Pyramid1の実現可能性を時間軸で支える
- 現地大手2社の経営統合により市場が流動化しており、新規参入の窓口が12ヶ月以内に閉じる見通し
- 2025年施行の現地EC規制緩和により、外資系プラットフォームの出店要件が大幅に簡素化された
- 競合A社が同市場への参入を発表しており、先行しなければ差別化ポイントを失う

**Pyramid 3（最下段）: 実行体制の整備完了**
← 「実際にできるのか」を裏付ける基盤的根拠
- 昨年設立した海外事業準備室に現地経験者15名を配置済みで、即時立上げが可能
- 国内EC基盤のマイクロサービス化が完了しており、海外向け機能の追加開発は6ヶ月で対応可能
- 現地物流パートナー3社との基本合意が締結済みで、フルフィルメント体制が確保されている

**なぜこれが良いピラミッドか:**
1. Pyramid1が「やる価値がある（収益）」→ Pyramid2が「今やる理由がある（タイミング）」→ Pyramid3が「やれる体制がある（実行力）」という**階層的な論理フロー**
2. 上位の段が「So What?」に答え、下位の段がそれを支えている
3. 各段が独立したMECEな観点（価値・時期・体制）をカバーしている

---

### ❌ 悪い例（よくある間違い）

**Main Message（NG）**
海外EC事業を検討すべき ← 「検討」は曖昧。「立ち上げるべき」のように具体的にする

**Pyramid 1: 市場が大きい**
**Pyramid 2: 技術力がある**
**Pyramid 3: 競合が少ない**

**なぜこれが悪いピラミッドか:**
- 3つの論点が**すべて同じ抽象度の並列な根拠**であり、階層関係がない
- 「市場が大きい」と「競合が少ない」はどちらも市場環境の話で、MECEではない
- 上位が下位を支える構造（「市場が大きい」→「だから技術力がある」？）にならない
- これはピラミッドではなく、SCRのSituation的な列挙に近い

## テーマ：DX推進の組織改革（5段ピラミッド）

### ✅ 良い例

**Main Message**
全社横断のDX推進体制を構築し、3年以内にコア業務の80%をデジタル化すべき

**Chart Title**
DX推進の組織改革方針

**Pyramid 1（最上段）: 経営層のコミットメント確保**
← 全体のDX推進を可能にする最上位の前提条件
- CEO直轄のDX推進室を設置し、意思決定スピードを従来の3倍に引き上げる
- 取締役会で四半期ごとにDX進捗をレビューする仕組みを導入する

**Pyramid 2: デジタル人材の確保・育成**
← 経営コミットメントの下で最初に整備すべきリソース
- 外部からDX専門人材を20名採用し、各事業部に配置する
- 全社員向けデジタルリテラシー研修を年2回実施し、基礎スキルを底上げする

**Pyramid 3: レガシーシステムの刷新**
← 人材が整った上で取り組むべきインフラ整備
- 基幹システムをクラウドネイティブに移行し、運用コストを40%削減する
- API基盤を整備し、部門間のデータ連携を自動化する

**Pyramid 4: データ活用基盤の構築**
← システム刷新を前提としたデータ戦略
- 全社統合データレイクを構築し、リアルタイム分析を可能にする
- BIツールを全管理職に展開し、データドリブンな意思決定文化を醸成する

**Pyramid 5（最下段）: 業務プロセスの自動化**
← 上記すべてが整った上で実現する最終的な成果
- RPA導入により定型業務の60%を自動化し、年間1,200時間の工数を削減する
- ワークフロー管理ツールを統一し、承認プロセスのリードタイムを半減する

**なぜこれが良いピラミッドか:**
1. 「戦略→人材→基盤→データ→業務」と**抽象度が高い順に段を構成**
2. 各段は前段を前提条件として成立する**依存関係**がある
3. 最上段（経営コミットメント）が全体を律し、最下段（業務自動化）は最も具体的な成果
