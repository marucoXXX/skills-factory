# Pyramid Structure PowerPoint ジェネレーター

Barbara Mintoの**Pyramid Principle**に基づき、メインメッセージを頂点として根拠・論点を階層的に整理し、ピラミッド図のPowerPointスライドを生成するスキル。

**PowerPointネイティブオブジェクト動的生成方式**により、**3〜7段の任意の段数**に対応する。すべてのテキスト・図形がPowerPoint上で直接編集可能。

---

## ピラミッドストラクチャーとは

McKinsey出身のBarbara Mintoが提唱したPyramid Principleの核心。結論（メインメッセージ）を頂点に置き、それを支える根拠をピラミッド状に階層化して示す論理構造。上位の主張は下位の根拠によって支えられ、聴衆は「なぜそう言えるのか？」を段階的に理解できる。

### 段数ガイドライン

| 段数 | 各段の推奨コメント数 | 用途 |
|---|---|---|
| **3段** | 各3コメント | 論点が3つ程度の場合。最も簡潔で伝わりやすい |
| **4段** | 各2〜3コメント | 論点が4つある場合。3段では説明不足の場合 |
| **5段** | 各2コメント | 論点が5つある場合。より多角的な根拠を示す |
| **6〜7段** | 各2コメント | 段階的な実行ステップや多層的な論理構造の場合 |

### 段数選択ルール

- **デフォルトは3段**。根拠・論点が3つで十分な場合に使用
- **4〜5段**: 根拠が4つ以上ある場合、または議事録から4〜5つの論点が抽出される場合
- **6〜7段**: 段階的プロセス（戦略→人材→基盤→データ→業務→改善）のような依存関係の連鎖を表現する場合
- ユーザーが段数を明示した場合はその指定に従う

## 各要素の定義と記述ルール

| 要素 | 定義 | ポイント |
|------|------|------|
| **Main Message** | ピラミッド全体の結論。読み手にアクションを促す一文。最大70文字 | 必ず「〜すべき」で締める |
| **Chart Title** | スライドのサブタイトル。ピラミッドのテーマや文脈を端的に示す短いフレーズ（10〜20文字） | Main Messageを補足する文脈を示す |
| **Pyramid N（段タイトル）** | 各段の論点・主張を端的に示すタイトル（10〜25文字） | 上位の段ほど抽象度が高く、下位は具体的 |
| **Pyramid N-Comment（コメント）** | 各段の詳細説明 | 段タイトルを裏付けるファクトや説明 |

### 記述ルール

- **Main Message**: 最大70文字。ピラミッド全体の結論を一言で総括し、必ず「〜すべき」で終える。
- **Chart Title**: 10〜20文字程度。Main Messageのテーマを補足する文脈フレーズ。
- **段タイトル（Pyramid N）**: 各段の主張を端的に表す。10〜25文字。
- **コメント（Pyramid N-Comment）**: 段タイトルを裏付ける具体的なファクト・データ・説明。1コメントあたり30〜60文字程度。
- **ピラミッドの上位→下位の論理**: 各段は**階層的な論理関係**を持つ。最上段がMain Messageの「So What?」に最も直接的に答え、下位の段はそれを成立させる条件・裏付けを示す。

## 入力パターンと処理フロー

### パターンA：議事録・文字起こしが入力された場合

**いきなりPowerPointを作成しない**

1. **Step 1: ピラミッド構造をドラフトする**（後述の抽出ガイドラインに従う）
2. **Step 2: Markdownでユーザーに提示**し、確認・修正を求める
3. **Step 3: ユーザーの承認後**、PowerPointを生成する

### パターンB：ピラミッド構造のテキストが入力された場合

1. **Step 1: 内容を確認**し、段数・コメント数が揃っているか確認
2. 不足・曖昧な点があれば修正提案を行う
3. **Step 2: 確認後、PowerPointを生成する**

---

## Step 1: 議事録からピラミッド構造を抽出する

### Main Messageの策定

議事録全体を俯瞰し、**最も重要な結論・提言**を一文にまとめる。
- 「何をすべきか」を端的に表現する
- 必ず「〜すべき」で終える
- 各段の根拠によって支えられる主張であること

### 段タイトルの抽出

議事録から**結論を支える主要な論点・根拠**を抽出し、**階層的な論理関係**で並べる：
- Pyramid1: Main Messageの核心に直結する最重要の根拠（「So What?」への直接回答）
- Pyramid2: Pyramid1を成立させる条件や、行動の緊急性を裏付ける根拠
- Pyramid3以降: 実現可能性・基盤的事実など、徐々に具体的な根拠
- 各段が独立したMECEな観点をカバーしつつ、上位→下位で論理が流れること
- **注意**: 同じ抽象度の並列な根拠を列挙するのはピラミッド構造ではない

### コメントの抽出

各段タイトルを裏付ける**具体的なファクト・データ・説明**を抽出する：
- 数字・指標・具体的事実を優先
- 3〜4段ピラミッドは各段2〜3コメント、5〜7段ピラミッドは各段2コメントが目安
- 議事録に不足があれば、戦略コンサルタントとしてドラフトし「※ドラフト」と明記する

## Step 3: PowerPointの生成

### テンプレート

段数問わず共通の1テンプレートを使用する：

```bash
TEMPLATE="assets/pyramid-template-universal.pptx"
```

※ `assets/` はこのスキルフォルダ配下を指す。フルパスはスキルの設置場所に応じて読み替えること。

### 生成方式

**PowerPointネイティブオブジェクト動的生成方式**。テンプレートの Title / Chart Title にテキストを流し込み、ピラミッド図部分はスクリプトが python-pptx で以下のShapeを動的に生成・配置する：

- **PyramidTier_N**: 各段のRectangle（上段ほど幅が狭く、下段ほど幅が広い、中央揃え）。段ごとに濃→薄のグラデーション色。Rectangle内にラベルテキストを直接配置。
- **PyramidDetail_N**: 右側の詳細TextBox。タイトル（bold）＋コメント行。薄いグレー背景。
- **AccentLine**: 詳細カード左のアクセント縦線。

すべてPowerPoint上で直接テキスト・色・サイズ・位置を編集可能。

### ピラミッドデータのJSON化

ピラミッド内容を `/home/claude/pyramid_data.json` に以下の形式で保存する：

```json
{
  "pyramid_type": 4,
  "main_message": "AI基盤を全社導入し、業務効率を50%改善すべき",
  "chart_title": "AI基盤導入の判断フレームワーク",
  "pyramids": [
    {
      "title": "経営インパクトの大きさ",
      "comments": ["年間コスト削減効果は30億円規模", "意思決定スピードが平均3倍に向上"]
    },
    {
      "title": "市場競争力の維持",
      "comments": ["競合上位5社中4社が既にAI基盤導入済み", "導入遅延は顧客離反リスクに直結"]
    },
    {
      "title": "技術的実現可能性",
      "comments": ["既存クラウド基盤との統合が容易", "PoCで精度95%以上を達成済み"]
    },
    {
      "title": "組織の受容性確保",
      "comments": ["パイロット部門でのNPS+45を記録", "研修プログラムの体系化が完了"]
    }
  ]
}
```

- `pyramid_type`: 段数（3〜7）。`pyramids` 配列の要素数と一致させること
- `pyramids`: 配列の順序がピラミッドの上段→下段に対応。要素数=段数
- `comments`: 各段のコメントリスト。段数に応じて2〜3個を推奨

### スクリプト実行コマンド

```bash
python <スキルフォルダ>/scripts/fill_pyramid.py \
  --data /home/claude/pyramid_data.json \
  --template <スキルフォルダ>/assets/pyramid-template-universal.pptx \
  --output /mnt/user-data/outputs/PyramidStructure_output.pptx
```

### 出力確認

```bash
python -m markitdown /mnt/user-data/outputs/PyramidStructure_output.pptx
```

内容が正しく反映されているか確認し、ユーザーに提示する。

## 品質チェックリスト

PowerPoint生成後、以下を確認：

- [ ] Main Messageが70文字以内で「〜すべき」で終わっているか
- [ ] Chart Titleが10〜20文字程度で文脈を的確に示しているか
- [ ] 段数がユーザーの指定通りか
- [ ] 各段タイトルが10〜25文字で端的に主張を表しているか
- [ ] Pyramid1（最上段）が最も重要・抽象的な根拠になっているか
- [ ] 上位→下位の論理が一貫しているか
- [ ] PPTXのmarkitdown出力でMain Message / Chart Titleが正しく反映されているか
