# Gate Process PowerPoint ジェネレーター

戦略コンサルタントが使う**ゲートプロセス（ファネル図）**スライドを生成するスキル。複数のフィルター（選定基準）を段階的に適用し、候補を絞り込むプロセスを視覚化する。

---

## ゲートプロセスとは

複数のフィルター（ゲート）を段階的に通過させ、候補を絞り込む選定プロセスを視覚化するフレームワーク。M&Aターゲット選定、投資先スクリーニング、サプライヤー評価、事業ポートフォリオの優先順位付けなど、段階的な意思決定に広く使われる。

| 要素 | 定義 | ポイント |
|------|------|------|
| **Main Message** | スライド全体を総括し、読み手にアクションを促す一文。最大70文字 | 必ず「〜すべき」で締める |
| **Chart Title** | スライドのサブタイトル。ゲートプロセスのテーマや文脈を端的に示す短いフレーズ（10〜20文字） | Main Messageを補足する文脈を示す |
| **Filter（フィルター）** | 各ゲートの名称（選定基準名）。3つまたは5つ | ファネルの上から下に向かって順番に適用される |
| **Feature（特徴）** | 各フィルターの具体的な評価項目や条件。各フィルターにつき2つ | フィルターの内容を具体的に説明する |
| **Implication（意味合い）** | ゲートプロセス全体から導かれる示唆・提言。3つ | 選定結果の解釈やネクストアクション |

### 記述ルール

- **Main Message**: 最大70文字。ゲートプロセス全体を一言で総括し、必ず「〜すべき」で終える。ユーザーが指定した場合はそのまま使用、指定がない場合はドラフトして確認を取る。
- **Chart Title**: 10〜20文字程度。Main Messageのテーマを補足する文脈フレーズ。ユーザーが指定しない場合はドラフトして確認を取る。
- **フィルターは3つまたは5つ**（テンプレートに合わせて選択）
- **各フィルターには2つのFeature**を記述する
- **Featureのフォーマット**: 具体的な評価基準・条件を簡潔に記述（1行に収まる長さ、目安20〜40文字）
- **Implicationは3つ**記述する
- **Implicationのフォーマット**: 選定結果から導かれる具体的な示唆・提言（目安30〜60文字）
- フィルターはファネル上部（広い）から下部（狭い）に向かって、徐々に厳しくなる基準順にする

## 入力パターンと処理フロー

### パターンA：議事録・文字起こしが入力された場合

**いきなりPowerPointを作成しない**

1. **Step 1: ゲートプロセスのドラフトを作成する**（後述の抽出ガイドラインに従う）
2. **Step 2: Markdownでユーザーに提示**し、確認・修正を求める
3. **Step 3: ユーザーの承認後**、PowerPointを生成する

### パターンB：フィルター条件が整理されたテキストが入力された場合

1. **Step 1: 内容を確認**し、フィルター数（3または5）、各Featureが2つずつあるか確認
2. 不足・曖昧な点があれば修正提案を行う
3. **Step 2: 確認後、PowerPointを生成する**

---

## Step 1: 議事録からゲートプロセスを抽出する

### フィルターの抽出

議事録から**段階的な選定基準・絞り込み条件**を拾う。以下のような表現を探す：
- 「まず〜で絞り込む」「次に〜を基準に」「最終的に〜で判断する」
- 「第一段階は〜」「スクリーニング基準として〜」
- 定量的な閾値や定性的な評価軸

フィルターの順序は**広い基準（足切り）→ 狭い基準（最終選定）**にする。

### Featureの抽出

各フィルターについて**具体的な評価項目や条件**を2つずつ拾う：
- 「具体的には〜と〜を見る」「〜の観点と〜の観点」
- 数値基準、定性基準、チェックポイント

### Implicationの抽出・ドラフト

議事録に意味合いが明示されていれば抽出。**明示されていない場合は、戦略コンサルタントとして論理的にドラフトする**：
- ゲートプロセスの結果として何が言えるか
- 次にどのようなアクションを取るべきか
- ドラフトした場合は「※ドラフト」と明記する

## Step 2: ゲートプロセスのMarkdown出力フォーマット

ユーザーに確認を求める際は、以下のフォーマットで出力する：

```markdown
## ゲートプロセス整理結果

**Main Message（※ドラフト）**
〜すべき（最大70文字）

**Chart Title（※ドラフト）**
〜のテーマ

### フィルター構成（3フィルター / 5フィルター）

**Filter 1: フィルター名**
- Feature 1: 具体的な評価項目・条件
- Feature 2: 具体的な評価項目・条件

**Filter 2: フィルター名**
- Feature 1: 具体的な評価項目・条件
- Feature 2: 具体的な評価項目・条件

**Filter 3: フィルター名**
- Feature 1: 具体的な評価項目・条件
- Feature 2: 具体的な評価項目・条件

（5フィルターの場合はFilter 4, 5も同様）

### 意味合い（Implications）
1. 意味合い1
2. 意味合い2
3. 意味合い3（※ドラフト）
```

確認メッセージ例：
> 上記のゲートプロセス整理でよろしいでしょうか？Main MessageとChart Titleも含めて修正があればお知らせください。確認後にPowerPointを生成します。

## Step 3: PowerPointの生成

### テンプレートの選択

フィルター数に応じてテンプレートを選択する：

- **3フィルター** → `assets/gate-process-3.pptx`
- **5フィルター** → `assets/gate-process-5.pptx`

```bash
# 3フィルターの場合
TEMPLATE="/mnt/skills/organization/gate-process-pptx/assets/gate-process-3.pptx"

# 5フィルターの場合
TEMPLATE="/mnt/skills/organization/gate-process-pptx/assets/gate-process-5.pptx"
```

テンプレートを初めて使う前に、以下で構造を確認すること：

```bash
pip install "markitdown[pptx]" -q --break-system-packages
python -m markitdown <TEMPLATE_PATH>
```

テンプレートのShape名は実際に確認してからマッピングすること。`references/template-mapping.md` に対応表を記載している。

### ゲートプロセスデータのJSON化

データを `/home/claude/gate_process_data.json` に以下の形式で保存する：

```json
{
  "main_message": "〜すべき（最大70文字）",
  "chart_title": "〜のテーマ（10〜20文字）",
  "filters": [
    {
      "name": "フィルター1の名前",
      "feature1": "Feature 1の説明",
      "feature2": "Feature 2の説明"
    },
    {
      "name": "フィルター2の名前",
      "feature1": "Feature 1の説明",
      "feature2": "Feature 2の説明"
    },
    {
      "name": "フィルター3の名前",
      "feature1": "Feature 1の説明",
      "feature2": "Feature 2の説明"
    }
  ],
  "implications": [
    "意味合い1",
    "意味合い2",
    "意味合い3"
  ]
}
```

5フィルターの場合はfilters配列に5つの要素を入れる。

### スクリプト実行コマンド

```bash
python /mnt/skills/organization/gate-process-pptx/scripts/fill_gate_process.py \
  --data /home/claude/gate_process_data.json \
  --template <TEMPLATE_PATH> \
  --output /mnt/user-data/outputs/GateProcess_output.pptx
```

### 出力確認

```bash
python -m markitdown /mnt/user-data/outputs/GateProcess_output.pptx
```

内容が正しく反映されているか確認し、ユーザーに提示する。

## 品質チェックリスト

PowerPoint生成後、以下を確認：

- [ ] Main Messageが70文字以内で「〜すべき」で終わっているか
- [ ] Chart Titleが10〜20文字程度で文脈を的確に示しているか
- [ ] フィルター数が3つまたは5つで、テンプレートと一致しているか
- [ ] 各フィルターにFeatureが2つずつあるか
- [ ] フィルターの順序が「広い基準→狭い基準」になっているか
- [ ] Implicationが3つあり、具体的な示唆になっているか
- [ ] PPTXのmarkitdown出力でプレースホルダーが残っていないか
