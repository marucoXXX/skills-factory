# データアベイラビリティ PowerPoint ジェネレーター

企業調査レポート（BDD・競合分析・M&A・新規参入調査等）の冒頭または末尾に配置する、
**調査の網羅度と制約を透明に示す1枚スライド**を生成するスキル。

戦略コンサルティングファームや投資銀行が作成する調査レポートでは、
**「何を調べたか」「何が取れなかったか」「どういう制約があるか」を明示することが信頼性の基盤**。
このスライドは、クライアントが調査結果の前提を正しく理解するための必須ドキュメント。

---

## スライド構成

| セクション | 位置 | 内容 |
|---|---|---|
| **メインメッセージ** | 最上部 | 調査全体の網羅度と制約を1文で。最大70文字 |
| **チャートタイトル** | メインメッセージ直下 | 「調査のデータアベイラビリティ：○○」 |
| **データ取得カバレッジ** | 左側（約8.0in） | カテゴリ×項目×ステータス×データソースのテーブル |
| **調査上の留意事項・制約** | 右側（約4.4in） | 薄イエロー背景+⚠マーク付きブレット項目 |
| **出典・調査期間** | 下部 | 情報ソースと調査実施期間の記載 |

### 左側: カバレッジテーブル

**テーブル構造:**
- **カテゴリ行**: 濃紺文字＋薄紺背景（#E8EEF4）で全列マージ、カテゴリ名（例：「対象会社」「市場環境」）
- **項目行**: 4列構成
  - 項目（左、左寄せ）
  - ステータス記号（中央）
  - ステータスラベル（色付き Bold）
  - データソース（右、左寄せ）
- **ゼブラストライピング**: 項目行は薄灰/白の交互（カテゴリ内でリセット）

**ステータス記号の種類:**

| 記号 | ラベル | 色 | 意味 |
|---|---|---|---|
| ✓ | 取得済 | 濃緑 #1B7A3B | 必要な情報を完全に取得 |
| △ | 一部取得 | オレンジ #DA7A2D | 一部のみ取得、または推計値で補完 |
| ✗ | 未取得 | 赤 #C03A3A | 公開情報では取得できず、追加調査が必要 |

### 右側: 制約事項パネル

- 薄イエロー背景（#FFF9E8）+ オレンジ枠線で注意喚起
- 各項目は ⚠マーカー付きのブレット項目
- 非上場競合データの不在、推計値の使用、言語的制約などを列挙

## 入力パターンと処理フロー

### パターンA：調査がこれから始まる場合

**いきなりPowerPointを作成しない**

1. **Step 1: 調査範囲を定義**（どのカテゴリ・項目を調べるか）
2. **Step 2: 各項目の取得見込みを整理**（✓/△/✗の予測）
3. **Step 3: 制約事項を整理**
4. **Step 4: Markdownでユーザーに提示**し、確認
5. **Step 5: 承認後**、PowerPointを生成する

### パターンB：調査完了後のレポーティング

1. **Step 1: 実際の取得状況をカテゴリ別に整理**
2. **Step 2: 制約事項を列挙**
3. **Step 3: Markdownでユーザーに提示**
4. **Step 4: 承認後**、PowerPointを生成する

### 推奨カテゴリ構成

企業調査の場合の典型的な4カテゴリ:

1. **対象会社**: 会社概要、事業ポートフォリオ、財務データ、戦略、顧客基盤
2. **マクロ環境**: 政策、経済、社会、技術トレンド
3. **市場環境**: 市場規模、成長率、シェア、セグメント細分化、市場予測
4. **競合**: 各競合の財務・事業情報、戦略、顧客

各カテゴリ内の項目は **2〜5個**を推奨。多すぎるとテーブルがはみ出すため注意。

## JSONデータ仕様

`{{WORK_DIR}}/data_availability_data.json` に以下の形式で保存する：

```json
{
  "source": "調査期間：2026年4月1日〜4月15日、公開情報のみ使用（Web検索・公式IR・業界レポート）",
  "main_message": "公開情報を中心に主要分析項目をほぼ網羅したが、非上場競合の財務データと一部の事業セグメント詳細は取得に制約あり",
  "chart_title": "調査のデータアベイラビリティ：対象会社競合分析",
  "categories": [
    {
      "name": "対象会社",
      "items": [
        {"label": "会社概要・沿革", "status": "complete", "source": "公式HP、EDINET"},
        {"label": "財務データ", "status": "complete", "source": "有報（過去5年）"},
        {"label": "主要顧客・取引先", "status": "partial", "source": "公式HP（一部のみ開示）"}
      ]
    },
    {
      "name": "競合",
      "items": [
        {"label": "上場競合3社の情報", "status": "complete", "source": "各社IR"},
        {"label": "非上場競合の財務", "status": "partial", "source": "TSR（連結は不明）"},
        {"label": "競合の顧客戦略", "status": "missing", "source": "公開情報では取得困難"}
      ]
    }
  ],
  "constraints": [
    "非上場競合2社の正確な財務データは取得不可",
    "市場シェアは2024年実績、2025年は推計ベース",
    "対象会社の内部戦略は追加ヒアリングが必要"
  ],
  "left_panel": {
    "section_title": "データ取得カバレッジ"
  },
  "right_panel": {
    "section_title": "調査上の留意事項・制約"
  }
}
```

### JSONフィールド仕様

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `source` | string | 任意 | 調査期間・情報源の記載 |
| `main_message` | string | 必須 | メインメッセージ（最大70文字） |
| `chart_title` | string | 任意 | チャートタイトル |
| `categories` | array | 必須 | カテゴリ定義配列（2〜5カテゴリ推奨） |
| `categories[].name` | string | 必須 | カテゴリ名 |
| `categories[].items` | array | 必須 | 項目配列（2〜5項目/カテゴリ推奨） |
| `categories[].items[].label` | string | 必須 | 項目名 |
| `categories[].items[].status` | string | 必須 | ステータス（下記参照） |
| `categories[].items[].source` | string | 任意 | データソース |
| `constraints` | array | 任意 | 制約事項ブレット配列（3〜6項目推奨） |
| `left_panel.section_title` | string | 任意 | 左セクションタイトル |
| `right_panel.section_title` | string | 任意 | 右セクションタイトル |

### `status` フィールドの受付値

| 意味 | 受付値 |
|---|---|
| 取得済 | `"complete"`, `"done"`, `"full"`, `"取得済"`, `"取得済み"`, `"✓"`, `"yes"`, `"ok"` |
| 一部取得 | `"partial"`, `"part"`, `"一部"`, `"△"`, `"limited"` |
| 未取得 | `"missing"`, `"none"`, `"no"`, `"✗"`, `"x"`, `"未取得"`, `"n/a"` |

### テーブル行数の上限

- **推奨合計行数**: 10〜15行（カテゴリ行+項目行）
- **最大**: 18行程度（それ以上はフォント縮小を検討 or 項目統合）
- 行数が多すぎると項目が読めなくなるため、項目を**適切にグループ化**する

## 品質チェックリスト

- [ ] メインメッセージが調査全体の網羅度・制約を1文で表しているか
- [ ] カテゴリ数が2〜5個の適切な範囲か
- [ ] 各カテゴリの項目数が2〜5個で、合計行数が15行以内に収まっているか
- [ ] 各項目にステータス（✓/△/✗）とデータソースが記載されているか
- [ ] ステータスの色分けが視覚的に分かりやすいか
- [ ] 制約事項が3〜6項目の適切な数で、薄イエロー背景に表示されているか
- [ ] 出典に調査期間（From-To）と情報源が記載されているか
- [ ] テーブルが出典と重なっていないか

## 注意事項

- **配置位置**: 通常はエグゼクティブサマリー直後、または末尾のAppendixに配置
- **透明性の原則**: 「✗未取得」を隠さず明示することで、調査の信頼性が向上する
- **ソース記載**: 各項目のソースは具体的に（「公式HP」ではなく「EDINET有価証券報告書」等）
- **制約事項の書き方**:
  - 具体的な企業名・数値を挙げる（「非上場競合X社・Y社」等）
  - 代替手段を示す（「推計値で補完」「TSRで補完」等）
  - 追加調査の方向性を示す（「ヒアリング要」等）
- **項目のグループ化**: 「財務データ（売上・営業利益・ROE等）」のように関連項目をまとめて1行にする

## 企業調査レポートでの使用例

### 配置パターン1: 冒頭に配置（透明性重視）

```
1. エグゼクティブサマリー
2. 調査のデータアベイラビリティ ← 本スキル（前提を最初に示す）
3. 対象会社プロファイル
...
```

### 配置パターン2: 末尾Appendixに配置（詳細を最後に）

```
1. エグゼクティブサマリー
2. 対象会社プロファイル
...
10. 各競合プロファイル
11. Appendix: 調査のデータアベイラビリティ ← 本スキル
```

### 用途別のカテゴリ構成例

**BDD（ビジネスDD）の場合:**
- 対象会社（会社概要・事業・財務・戦略・顧客）
- マクロ環境
- 市場環境
- 競合
- リスク要因

**M&Aターゲット評価の場合:**
- ターゲット企業
- バリュエーション関連データ
- 市場・業界
- シナジー可能性

**新規参入調査の場合:**
- マクロ環境
- 市場規模・成長性
- 既存プレイヤー
- 参入障壁・成功要因
