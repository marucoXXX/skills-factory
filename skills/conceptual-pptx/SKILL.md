# コンセプト図 PowerPoint ジェネレーター

複数のコンセプト（概念・要素）の関係性を楕円の重なりで視覚化し、戦略コンサルタントスタイルのPowerPointスライドを生成するスキル。

---

## コンセプト図とは

複数の概念・要素が互いにどう関係しているかを、楕円（Oval）の重なりで視覚的に表現するスライド。左側に楕円の重なり図、右側に各コンセプトの名前と詳細説明を配置する。

### テンプレート構成

| バリアント | コンセプト数 | 用途 |
|---|---|---|
| **Conceptual3** | 3つ | シンプルな構造。3要素の関係性を示す場合 |
| **Conceptual5** | 5つ | 複雑な構造。5要素の関係性を示す場合 |

### 各要素の定義

| 要素 | 定義 | ポイント |
|------|------|------|
| **Main Message** | コンセプト図全体を総括し、読み手にアクションを促す一文。最大70文字 | 必ず「〜すべき」で締める |
| **Chart Title** | スライドのサブタイトル。コンセプト図のテーマや文脈を端的に示す短いフレーズ（10〜20文字） | Main Messageを補足する文脈を示す |
| **Concept Name** | 各コンセプトの名称。短く明確な名前（2〜10文字） | 楕円内と右側テキストの両方に表示される |
| **Concept Detail（3版）** | 各コンセプトの説明文を2行で記述。詳細1と詳細2で異なる切り口（What/How、手段/効果など）を書き分ける | 各行1文、合計2文で具体的に書く |
| **Concept Detail（5版）** | 各コンセプトの説明文を1行で記述。そのコンセプトが何を意味するか、なぜ重要かを端的に記述 | 1文で具体的に書く |

### 記述ルール

- **Main Message**: 最大70文字。コンセプト図全体を一言で総括し、必ず「〜すべき」で終える。ユーザーが指定した場合はそのまま使用、指定がない場合はコンセプト内容をもとにドラフトして確認を取る。
- **Chart Title**: 10〜20文字程度。Main Messageのテーマを補足する文脈フレーズ。ユーザーが指定しない場合はドラフトして確認を取る。
- **Concept Name**: 2〜10文字。簡潔で明確な名前。楕円図と右側テキストの両方に表示されるため、短さが重要。
- **Concept Detail（3コンセプト版）**: 各コンセプトに2行の詳細を記述する。詳細1と詳細2は異なる切り口で書き分ける（例: What/How、手段/効果、現状/目標）。各行1文で端的に書く。
- **Concept Detail（5コンセプト版）**: 各コンセプトに1行の詳細を記述する。「〜であること」「〜を実現する仕組み」のように、コンセプトの本質を端的に表現する。

## テンプレート選択ルール

ユーザーの入力からコンセプト数を判断し、適切なテンプレートを自動選択する。

| コンセプト数 | テンプレート | 備考 |
|---|---|---|
| 3以下 | Conceptual3.pptx | 3つに満たない場合でも3テンプレートを使用し、必要に応じてコンセプトを追加提案 |
| 4〜5 | Conceptual5.pptx | 4つの場合も5テンプレートを使用し、5つ目のコンセプトを追加提案 |
| 6以上 | ユーザーに確認 | 5つに絞るか、複数スライドに分割するか相談する |

## 入力パターンと処理フロー

### パターンA：議事録・文字起こしが入力された場合

**いきなりPowerPointを作成しない**

1. **Step 1: コンセプトドラフトを作成する**（後述の抽出ガイドラインに従う）
2. **Step 2: Markdownでユーザーに提示**し、確認・修正を求める
3. **Step 3: ユーザーの承認後**、PowerPointを生成する

### パターンB：コンセプトが明確に指定された場合

1. **Step 1: 内容を確認**し、コンセプト数・名称・詳細が揃っているか確認
2. 不足・曖昧な点があれば修正提案を行う
3. **Step 2: 確認後、PowerPointを生成する**

---

## Step 1: 議事録からコンセプトを抽出する

### コンセプトの抽出方針

議事録や入力テキストから、**中核となる概念・要素**を特定する。以下のような表現を探す：

- 「〜という考え方」「〜のコンセプト」「〜が重要だ」「〜を柱にする」
- 繰り返し言及されるキーワードやテーマ
- 「〜と〜の関係」「〜を組み合わせる」「〜が〜を支える」

### コンセプト名の付け方

- **短く明確に**: 2〜10文字で本質を表現する
- **名詞形が望ましい**: 「デジタル変革」「顧客中心設計」「持続的成長」など
- **抽象度を揃える**: 一つだけ具体的すぎる/抽象的すぎるものがないように

### コンセプト詳細の書き方

**3コンセプト版（詳細2行）:**
- 詳細1と詳細2は**異なる切り口**で書き分ける
- 推奨パターン:
  - What + How（何をするか ＋ どう実現するか）
  - 手段 + 効果（具体的アクション ＋ 期待される成果）
  - 現状 + 目標（今の取り組み ＋ 目指す到達点）
- 各行1文で端的に書く（長すぎるとスライドに収まらない）

**5コンセプト版（詳細1行）:**
- そのコンセプトが**何を意味するか**を具体的に説明する
- **なぜ重要か**、**どう機能するか**を含める
- 1文で端的に書く

NG例：
- 「重要である」← 何が重要なのか不明
- 「推進する」← 具体性がない

OK例：
- 「AIとデータ分析を活用し、顧客行動の予測精度を従来比3倍に向上させる基盤」
- 「全社横断のプロジェクトチームにより、サイロ化した部門間の連携を実現する仕組み」

## Step 2: コンセプトのMarkdown出力フォーマット

ユーザーに確認を求める際は、以下のフォーマットで出力する：

```markdown
## コンセプト図 整理結果

**Main Message（※ドラフト）**
〜すべき（最大70文字）

**Chart Title（※ドラフト）**
〜のテーマ

**テンプレート**: Conceptual3（3コンセプト版・詳細2行） / Conceptual5（5コンセプト版・詳細1行）

### コンセプト一覧（3コンセプト版の場合）
1. **コンセプト名A**
   - 詳細1: 説明文（What / 手段 / 現状）
   - 詳細2: 説明文（How / 効果 / 目標）
2. **コンセプト名B**
   - 詳細1: 説明文
   - 詳細2: 説明文
3. **コンセプト名C**
   - 詳細1: 説明文
   - 詳細2: 説明文

### コンセプト一覧（5コンセプト版の場合）
1. **コンセプト名A**: 詳細の説明文
2. **コンセプト名B**: 詳細の説明文
3. **コンセプト名C**: 詳細の説明文
4. **コンセプト名D**: 詳細の説明文
5. **コンセプト名E**: 詳細の説明文
```

確認メッセージ例：
> 上記のコンセプト整理でよろしいでしょうか？Main MessageとChart Titleも含めて修正があればお知らせください。確認後にPowerPointを生成します。

## Step 3: PowerPointの生成

### テンプレートの参照

テンプレートはスキルの `assets/` ディレクトリにある:

```bash
TEMPLATE3="<skill_path>/assets/Conceptual3.pptx"
TEMPLATE5="<skill_path>/assets/Conceptual5.pptx"
```

`<skill_path>` はこのスキルがインストールされたディレクトリパス。

テンプレートを初めて使う前に、以下で構造を確認すること：

```bash
pip install "markitdown[pptx]" -q --break-system-packages
python -m markitdown <skill_path>/assets/Conceptual3.pptx
python -m markitdown <skill_path>/assets/Conceptual5.pptx
```

テンプレートのShape名は実際に確認してからマッピングすること。`references/template-mapping.md` に確認済みの対応表を記載してある。

### コンセプトデータのJSON化

コンセプト内容を `{{WORK_DIR}}/conceptual_data.json` に以下の形式で保存する：

**3コンセプト版（詳細2行）:**
```json
{
  "main_message": "〜すべき（最大70文字）",
  "chart_title": "〜のテーマ（10〜20文字）",
  "variant": 3,
  "concepts": [
    {"name": "コンセプト名A", "detail1": "詳細1の文章", "detail2": "詳細2の文章"},
    {"name": "コンセプト名B", "detail1": "詳細1の文章", "detail2": "詳細2の文章"},
    {"name": "コンセプト名C", "detail1": "詳細1の文章", "detail2": "詳細2の文章"}
  ]
}
```

**5コンセプト版:**
```json
{
  "main_message": "〜すべき（最大70文字）",
  "chart_title": "〜のテーマ（10〜20文字）",
  "variant": 5,
  "concepts": [
    {"name": "コンセプト名A", "detail": "詳細の説明文"},
    {"name": "コンセプト名B", "detail": "詳細の説明文"},
    {"name": "コンセプト名C", "detail": "詳細の説明文"},
    {"name": "コンセプト名D", "detail": "詳細の説明文"},
    {"name": "コンセプト名E", "detail": "詳細の説明文"}
  ]
}
```

### スクリプト実行コマンド

```bash
python <skill_path>/scripts/fill_conceptual.py \
  --data {{WORK_DIR}}/conceptual_data.json \
  --template3 <skill_path>/assets/Conceptual3.pptx \
  --template5 <skill_path>/assets/Conceptual5.pptx \
  --output {{OUTPUT_DIR}}/Conceptual_output.pptx
```

### 出力確認

```bash
python -m markitdown {{OUTPUT_DIR}}/Conceptual_output.pptx
```

内容が正しく反映されているか確認し、ユーザーに提示する。

## 品質チェックリスト

PowerPoint生成後、以下を確認：

- [ ] Main Messageが70文字以内で「〜すべき」で終わっているか
- [ ] Chart Titleが10〜20文字程度で文脈を的確に示しているか
- [ ] コンセプト数がテンプレートと一致しているか（3 or 5）
- [ ] すべてのコンセプト名が2〜10文字で簡潔か
- [ ] すべてのコンセプト詳細が具体的か
- [ ] 3コンセプト版の場合、各コンセプトに詳細が2行あり、異なる切り口で書き分けられているか
- [ ] 5コンセプト版の場合、各コンセプトに詳細が1行あるか
- [ ] PPTXのmarkitdown出力でプレースホルダーが残っていないか
