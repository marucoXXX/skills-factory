# コンセプト図の良い例・悪い例

## テーマ：DX推進の3つの柱（3コンセプト版・詳細2行）

### ✅ 良い例

**Main Message**
DX推進には「データ基盤」「組織変革」「顧客体験」の3つの柱を同時に強化すべき

**Chart Title**
DX推進の3つの柱

**コンセプト一覧**
1. **データ基盤**
   - 詳細1: 全社横断のデータレイクを構築し、リアルタイム分析基盤として事業判断のスピードを3倍に高める
   - 詳細2: BIツールを全事業部に展開し、データドリブンな意思決定文化を定着させる
2. **組織変革**
   - 詳細1: デジタル人材を各事業部に配置し、現場主導でDX施策を企画・実行できる体制を構築する
   - 詳細2: 経営層のDXリテラシー向上プログラムを導入し、トップダウンの推進力を確保する
3. **顧客体験**
   - 詳細1: オンライン・オフラインの顧客接点を統合し、パーソナライズされた体験を全チャネルで提供する
   - 詳細2: NPS指標を経営KPIに組み込み、顧客満足度を継続的にモニタリングする体制を整備する

### ❌ 悪い例

**コンセプト名が曖昧**
1. **デジタル**
   - 詳細1: デジタル化を推進する ← 抽象的すぎる
   - 詳細2: 良い感じにする ← 何も言っていない

**詳細1と詳細2の役割が不明確**
1. **データ基盤**
   - 詳細1: データを活用する ← 具体性ゼロ
   - 詳細2: データを活用して成果を出す ← 詳細1と同じことを言っている

## テーマ：新規事業の成功要因（5コンセプト版・詳細1行）

### ✅ 良い例

**Main Message**
新規事業を成功させるには5つの要素を有機的に連携させ、スピード感をもって実行すべき

**Chart Title**
新規事業成功の5要素

**コンセプト一覧**
1. **市場洞察**: ターゲット市場の未充足ニーズを定量・定性の両面から特定し、仮説を構築する
2. **プロダクト力**: MVPを90日以内にリリースし、顧客フィードバックを週次で反映する高速開発体制を構築する

4. **資金調達**: 段階的なマイルストーンを設定し、各フェーズで必要十分な資金を確保する
5. **パートナーシップ**: 自社にない技術・チャネル・ブランド力を持つパートナーとの戦略的提携を推進する

## 3コンセプト版の詳細2行の書き分け方

3コンセプト版では各コンセプトに2行の詳細を記述する。以下のような書き分けが効果的：

| パターン | 詳細1の役割 | 詳細2の役割 |
|---|---|---|
| What + How | 何をするか（施策の概要） | どう実現するか（具体的手段） |
| 現状 + 目標 | 現在の取り組み・状態 | 目指すべき到達点 |
| 手段 + 効果 | 具体的なアクション | 期待される成果・インパクト |

## Main Messageの書き方ガイド

### ルール
- 最大70文字
- 必ず「〜すべき」で終える
- コンセプト図全体を一言で総括する

### 良い例
- 「DX推進には技術・組織・顧客の3軸を同時に強化すべき」（28文字）
- 「持続的成長のために5つの戦略的要素を有機的に連携させるべき」（29文字）
- 「AI時代の競争優位はデータ・人材・プロセスの3層構造で築くべき」（31文字）

### 悪い例
- 「DXを推進すべき」← 何のコンセプトか不明、Main Messageとして情報不足
- 「3つの柱がある」← 「〜すべき」で終わっていない
- 「デジタルトランスフォーメーションの推進においては、データ基盤の構築と組織変革の実行と顧客体験の向上を三位一体で同時並行的に推進していくことが極めて重要であり、これらを統合的に実施すべき」← 70文字超過
