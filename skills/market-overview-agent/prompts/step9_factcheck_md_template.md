# Step 9: FactCheck_Report.md 雛形

`fact-check-reviewer` が出力した `fact_check_report.json` を整形して、最終納品物として
`{{OUTPUT_DIR}}/FactCheck_Report_<market_name>_<date>.md` に保存する。

クライアントへ「PPTX デッキの数値・固有名詞がどのソースで裏取りされ、どれが要確認か」を
透明に提示するための文書。

## 1. 概要

本レポートは、デッキ生成過程で Web 検索により収集した数値・シェア・市場規模・日付・
固有名詞などのクレーム（事実主張）を、別の Web 検索によって裏取りした結果を示す。

`severity=high` のクレームは PPTX 生成前にユーザー確認の上で修正・注釈付与済み。
本レポートは事後的なエビデンス記録として、クライアントが原典を辿れるようにする目的で
納品する。

### 検証結果の凡例

| 値 | 意味 |
|----|------|
| `confirmed` | 複数の信頼できるソースで主張と一致する値が確認できた |
| `single_source` | 単一ソースのみで確認。クロスチェックは未済 |
| `discrepancy` | 主張と異なる値が他ソースで見つかった |
| `not_found` | 裏取り検索で該当する情報が見つからなかった |
| `stale` | 確認できたが、原典が古く、より新しい情報がある可能性 |

### severity の凡例

| 値 | 意味 |
|----|------|
| `high` | 値の差が大きい（数値で20%以上、年度で2年以上のズレ等）。即修正対象 |
| `medium` | 単一ソース、または出典が一次情報でない |
| `low` | 一次情報で確認済み、または曖昧出典で誤差許容範囲 |

## 3. 自信度サマリー

```
High confidence  : XX 件 (一次情報・複数ソース確認)
Medium confidence: YY 件 (単一信頼ソース確認)
Low confidence   : ZZ 件 (推定・出典曖昧)
```

クライアントが追加裏取りを行う場合は、severity が medium 以上のクレームから優先的に
原典確認することを推奨する。

## 補足

- このレポートは PPTX デッキと並列に納品されるエビデンス文書である
- クライアントが原典を辿れるよう、出典URLは可能な限り恒久URL（PDFや報道発表）を選ぶ
- 検索結果が消えるリスクが高いソース（個人ブログ・SNS）は使用しない
- ファクトチェックスコープが `skip` の場合、本レポートには「Step 2.5 をスキップ」と記載のみ
- 個人情報・社名以外のセンシティブな情報は本レポートにも含めない

## 整形時の注意

1. `fact_check_report.json` の `flags[]` を `severity` 降順 → `slide_number` 昇順でソート
2. 表が長くなる場合は、`severity=high` のクレームを上に配置するか、別表に切り出す
3. `not_found` のクレームには「(検索結果なし)」と明示し、該当主張をデッキでどう扱ったか
   （例: 注釈付与 / そのまま掲載 / 削除）を「補足」列に必ず記載する
4. `discrepancy` の場合、検索で見つかった値も「補足」列に併記する（クライアントが判断できる
   情報を残す）
