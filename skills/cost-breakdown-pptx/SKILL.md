# コスト内訳推移 PowerPoint ジェネレーター

BDD（ビジネスデュー・ディリジェンス）において、外注費・仕入高・経費などの
コスト内訳推移を積み上げ棒グラフ＋折れ線グラフで1枚のPowerPointスライドに整理するスキル。

---

## スライド構成

| セクション | 位置 | 内容 |
|---|---|---|
| **メインメッセージ** | 最上部 | 最大70文字。「〜すべき」で締める |
| **チャート領域** | コンテンツエリア | 1チャート（フルワイド）または2チャート（左右並列） |
| **出典** | 左下 | 情報ソースの記載 |

### 各チャートの構成

| 要素 | 位置 | 内容 |
|---|---|---|
| **チャートタイトル** | チャート上部中央 | 「外注先別の外注費推移」等、下線付き |
| **単位ラベル** | タイトル下左寄せ | 「（単位：百万円、%）」 |
| **凡例** | 単位ラベル下 | 棒系列＋折れ線凡例。行折り返し対応 |
| **積み上げ棒グラフ** | チャート本体 | 先別・カテゴリ別の積み上げ。各セグメントに%ラベル |
| **折れ線グラフ** | チャート本体（第2軸） | 対売上高比率等。%ラベル付き |
| **トータルラベル** | 棒の上 | 合計金額（テキストボックス） |

### レイアウトモード

- **1チャートモード**: `charts`配列に1つ → フルワイド表示
- **2チャートモード**: `charts`配列に2つ → 左右並列表示

---

## 入力パターンと処理フロー

### パターンA：IM・対象会社資料・Excelデータが入力された場合

**いきなりPowerPointを作成しない**

1. **Step 1: コスト内訳データを抽出・整理する**
   - 外注先/仕入先ごとの金額と割合を期別に整理
   - 上位N社＋その他に集約
   - 対売上高比率を算出
2. **Step 2: Markdownでユーザーに提示**し、確認・修正を求める
3. **Step 3: ユーザーの承認後**、PowerPointを生成する

### パターンB：データが整理済みで入力された場合

1. **Step 1: 内容を確認**し、データの整合性を確認
2. **Step 2: 確認後、PowerPointを生成する**

---

## Step 3: PowerPointの生成

### JSONデータ仕様

`{{WORK_DIR}}/cost_breakdown_data.json` に以下の形式で保存する：

```json
{
  "main_message": "外注費は売上拡大に伴い増加しており、主要外注先への依存度と原材料仕入先の分散状況を評価すべき",
  "source": "対象会社提供資料",
  "charts": [
    {
      "title": "外注先別の外注費推移",
      "unit_label": "（単位：百万円、%）",
      "stacked_bars": [
        {"series_name": "A社", "color": "#4E79A7"},
        {"series_name": "B社", "color": "#59A14F"},
        {"series_name": "その他", "color": "#BAB0AC"}
      ],
      "line": {
        "series_name": "対売上高外注費比率",
        "color": "#4E79A7"
      },
      "data": [
        {
          "year": "21/6期",
          "bars": [14.4, 9.5, 39.1],
          "shares": [22.8, 15.0, 62.2],
          "line_value": 16.8,
          "total_label": "63"
        }
      ]
    }
  ]
}
```

### JSONフィールド仕様

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `main_message` | string | 必須 | メインメッセージ。最大70文字、「〜すべき」で締める |
| `source` | string | 任意 | 出典テキスト（左下に表示） |
| `charts` | array | 必須 | チャート定義の配列（1〜2要素） |
| `charts[].title` | string | 任意 | チャートタイトル（中央表示、下線付き） |
| `charts[].unit_label` | string | 任意 | 単位表記（例: "（単位：百万円、%）"） |
| `charts[].stacked_bars` | array | 必須 | 積み上げ棒グラフの系列定義（下から順） |
| `charts[].stacked_bars[].series_name` | string | 必須 | 系列名 |
| `charts[].stacked_bars[].color` | string | 任意 | 色コード（例: "#4E79A7"）。未指定時はデフォルトパレット |
| `charts[].line` | object | 任意 | 折れ線グラフの系列定義。省略可 |
| `charts[].line.series_name` | string | 必須 | 系列名 |
| `charts[].line.color` | string | 任意 | 色コード。デフォルト "#4E79A7" |
| `charts[].line.marker_size` | int | 任意 | マーカーサイズ。デフォルト 7 |
| `charts[].data` | array | 必須 | 期別データの配列（2〜5要素推奨） |
| `charts[].data[].year` | string | 必須 | 期ラベル（例: "21/6期"） |
| `charts[].data[].bars` | array | 必須 | 各系列の金額値（stacked_barsの順序に対応） |
| `charts[].data[].shares` | array | 必須 | 各系列の割合（%）。棒セグメント内に%ラベルとして表示 |
| `charts[].data[].line_value` | number | 任意 | 折れ線の値（%） |
| `charts[].data[].total_label` | string | 任意 | 棒の上に表示する合計ラベル。空文字で非表示 |

### %ラベル表示ルール

- `shares`配列の値が **2.0%以上** のセグメントにのみ白文字で%ラベルを表示
- 2.0%未満のセグメントはラベル非表示（棒が小さすぎるため）
- 表示形式: 小数点1桁（例: "22.8%"）

## チャートのバリエーション

### 折れ線なしパターン

`charts[].line` を省略すると、積み上げ棒グラフのみのチャートになる。

### 1チャートモード

`charts` 配列を1要素にすると、フルワイドの単一チャート表示になる。

### トータルラベルなし

`data[].total_label` を空文字にすると、合計ラベルが非表示になる。

## 品質チェックリスト

- [ ] メインメッセージが正しく表示されているか

- [ ] 積み上げ棒グラフの全データポイントが表示されているか

- [ ] トータルラベルが棒の上に正しく表示されているか
- [ ] 折れ線グラフが正しく表示されているか（設定時）
- [ ] 折れ線のデータラベルが%付きで表示されているか
- [ ] 凡例が各チャート上部に正しく表示されているか
- [ ] 2チャートモードで左右のチャートが重ならないか
- [ ] X軸の期ラベルが正しく表示されているか
- [ ] 出典が左下に表示されているか
- [ ] PowerPointネイティブオブジェクトで生成されており、人間が編集可能か
