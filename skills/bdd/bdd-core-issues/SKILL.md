# BDD コア論点マスター

業界横断のBDDコア論点を `assets/core_issues.json` で提供するアセットスキル。

## 全体構造

論点ツリーは2階層：
- **L1（カテゴリ）**: 10カテゴリ
- **L2（論点）**: 各L1配下に3〜5論点、合計41論点

### L1カテゴリ一覧

| ID | カテゴリ | 目的 |
|---|---|---|
| L1-01 | 市場環境 | 対象会社が戦う市場の魅力度・成長性・構造 |
| L1-02 | 競争環境 | 業界内の競争構造と相対ポジション |
| L1-03 | ビジネスモデル | どう稼ぐか・どう価値を生むか |
| L1-04 | 商流・顧客 | 顧客との関係・販売構造の安定性とリスク |
| L1-05 | オペレーション | 事業基盤の競争力とボトルネック |
| L1-06 | 組織・人材 | 組織能力と人的資産・経営の質 |
| L1-07 | 財務 | 業績の実態・収益性・財務健全性 |
| L1-08 | 成長戦略 | 成長ストーリーと実現可能性 |
| L1-09 | リスク | 事業価値を毀損し得るリスク要因 |
| L1-10 | ESG・規制 | 業界規制とサステナビリティ課題への対応 |

## 各L2論点のフィールド

```json
{
  "id": "L2-XX-XX",
  "name": "論点名",
  "key_questions": ["この論点で答えるべき具体的な問い", "..."],
  "typical_verification_approach": "典型的な検証アプローチ",
  "typical_data_sources": ["典型的なデータソース"],
  "default_priority": "High | Mid | Low"
}
```

## ID体系

- L1: `L1-01` 〜 `L1-10`
- L2: `L2-{L1番号}-{L2連番}` (例: `L2-01-03` = L1-01の3番目)

## 使い方

このスキル自体には実行スクリプトがない。`assets/core_issues.json` をプロジェクトの `issues.json` にコピーしてプロジェクトを開始する役割は `bdd-init` スキルが担う。

ユーザーがコア論点の中身を確認したい場合は、`assets/core_issues.json` を直接読んで内容を提示すれば良い。

## アップデートポリシー

このマスターは「業界横断で本当に重要な論点」のみを保持する。プロジェクト固有の論点は `issues.json` 側で追加する。マスター自体の更新は、複数プロジェクトで共通して必要だと判明した論点があった場合のみ行う。
