# プロジェクト体制図の良い例・悪い例

## テーマ：DX推進プロジェクト

### ✅ 良い例

**Main Message**
DX推進は経営直轄の3WG体制で各領域の専門家を配置し、6ヶ月で基盤構築を完了すべき

**Chart Title**
DX推進プロジェクト体制

**体制**
- Project Sponsor: 取締役CTO 山田太郎
- Project Owner: DX推進部長 鈴木一郎
- PMO: 経営企画部 DX推進室

**ワーキンググループ**
- WG1 データ基盤構築: 佐藤花子、田中次郎、高橋三郎
- WG2 業務プロセス改革: 伊藤四郎、渡辺五郎、小林六子、加藤七郎
- WG3 チェンジマネジメント: 吉田八郎、山本九子

**意味合い（Implications）**
1. CTO直轄体制により経営レベルでの迅速な意思決定が可能
2. PMOを経営企画部に設置し、全社横断的な調整機能を確保
3. WG1のデータ基盤構築が全体のクリティカルパスであり、最優先でリソースを配分
4. WG2とWG3は並行推進するが、WG2の業務設計完了後にWG3の本格展開を開始
5. 各WGリーダーは週次でPMOに進捗報告し、課題のエスカレーションルートを明確化

---

### ❌ 悪い例（よくある間違い）

**Main Message（NG）**
DX推進の体制図 ← 「〜すべき」で終わっていない。単なるタイトルになっている

**Main Message（NG）**
しっかりとした体制でDXを進めるべき ← 抽象的すぎる。何がどう体制なのかが不明

**体制（NG: 具体性がない）**
- Project Sponsor: 経営層 ← 誰か特定すべき
- Project Owner: 担当部門 ← 個人名が必要
- PMO: 未定 ← 未定ならその旨を明記し、いつまでに決めるかを意味合いに記載

**意味合い（NG）**
1. この体制で進める ← 意味合いになっていない（単なる事実の再述）
2. 各WGが頑張る ← 抽象的すぎる
3. 連携が重要 ← 何と何がどう連携するのかが不明

---

## テーマ：基幹システム刷新プロジェクト（5WG体制）

### ✅ 良い例

**Main Message**
基幹システム刷新は5WG体制で段階移行を行い、業務停止リスクをゼロにすべき

**Chart Title**
基幹システム刷新プロジェクト推進体制

**体制**
- Project Sponsor: 代表取締役社長 中村一夫
- Project Owner: 情報システム本部長 木村二郎
- PMO: アクセンチュア PMOチーム

**ワーキンググループ**
- WG1 要件定義: 松本三郎、井上四子、森五郎
- WG2 インフラ構築: 清水六郎、池田七子
- WG3 データ移行: 橋本八郎、石川九子、前田十郎、藤田十一子
- WG4 テスト・品質管理: 後藤十二郎、近藤十三子
- WG5 運用・保守設計: 村上十四郎、坂本十五子、遠藤十六郎

**意味合い（Implications）**
1. 社長直轄体制により、部門間の利害調整を経営トップの意思決定で迅速に解消
2. 外部コンサルにPMOを委託し、社内政治に影響されない客観的な進捗管理を実現
3. WG3データ移行が最大のリスク領域であり、4名体制で冗長性を確保
4. WG4テストチームを独立設置し、開発チームとの利益相反を排除
5. WG5を初期段階から参画させ、運用設計を開発と並行して進めることで引き継ぎリスクを低減
