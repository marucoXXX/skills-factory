# 事業ポートフォリオ PowerPoint ジェネレーター

対象会社（または競合）の事業セグメント別の売上構成と収益性を1枚のPowerPointスライドに整理するスキル。
BDD（ビジネスデュー・ディリジェンス）や競合分析において、対象会社の「事業ミックス」や「成長/利益ドライバー」を可視化する。

---

## スライド構成

| セクション | 位置 | 内容 |
|---|---|---|
| **メインメッセージ** | 最上部 | 最大70文字。「〜すべき」で締める |
| **チャートタイトル** | メインメッセージ直下 | 「事業ポートフォリオ推移：○○社」 |
| **セグメント別売上推移** | 左側 | ネイティブ積み上げ棒チャート（年別、セグメント色分け、データラベル付き） |
| **セグメント別サマリー** | 右側 | サマリーテーブル（セグメント、売上、構成比、CAGR、営業利益率） |
| **出典** | 左下 | 情報ソースの記載 |

### 左側: セグメント別売上推移（積み上げ棒チャート）

- ネイティブ PowerPoint チャート（XL_CHART_TYPE.COLUMN_STACKED）
- 各セグメントは指定色または自動割り当て色で表示
- データラベル: 各セグメントの値を棒の中央に白文字で表示（Bold、9pt）
- 凡例: チャート下部に配置
- Y軸は非表示、目盛線なし（シンプルな見た目）
- Y軸最大値は合計×1.20で自動パディング（データラベルが切れないよう）

### 右側: セグメント別サマリーテーブル

- ヘッダー行: 濃紺背景（#2E4A6B）、白文字、中央揃え
- データ行: 奇数行は薄灰背景（#F2F2F2）、偶数行は白背景（ゼブラストライピング）
- セグメント名列: Bold、先頭にセグメント色の「■」マーカー（チャートと色で連動）
- 数値列: 右揃え
- 柔軟なヘッダー対応: 「セグメント」「売上高」「構成比」「CAGR」「営業利益率」などのキーワードを自動マッピング

### 年度範囲のルール

- **End year**: 実行年の前年（2026年実行なら2025年）
- **Start year**: そこから3〜5年遡る（セグメント情報の開示が揃っている範囲）
- **CAGR**: データ配列の最初と最後の年から計算してテーブルに表示（ユーザーが事前計算）

## 入力パターンと処理フロー

### パターンA：有報・決算短信のセグメント情報が入力された場合

**いきなりPowerPointを作成しない**

1. **Step 1: セグメント情報を抽出・整理する**
2. **Step 2: Markdownでユーザーに提示**し、確認・修正を求める
3. **Step 3: ユーザーの承認後**、PowerPointを生成する

### パターンB：データが整理済みで入力された場合

1. **Step 1: 内容を確認**し、項目が揃っているか確認
2. **Step 2: 確認後、PowerPointを生成する**

## JSONデータ仕様

`/home/claude/business_portfolio_data.json` に以下の形式で保存する：

```json
{

  "main_message": "当社は自動車事業が売上の80%を占めるが、金融・その他事業が利益率の底上げに貢献している点を評価すべき",
  "chart_title": "事業ポートフォリオ推移：トヨタ自動車株式会社（連結）",
  "segments": [
    {"name": "自動車事業", "color": "#4E79A7"},
    {"name": "金融事業", "color": "#F28E2B"},
    {"name": "その他事業", "color": "#59A14F"}
  ],
  "chart": {
    "section_title": "セグメント別売上高推移",
    "unit_label": "（単位：十億円）",
    "years": ["FY2020", "FY2021", "FY2022", "FY2023", "FY2024"],
    "values": {
      "自動車事業": [25200, 27100, 32700, 38800, 41500],
      "金融事業": [2100, 2200, 2600, 3300, 3800],
      "その他事業": [980, 1000, 1100, 1250, 1400]
    }
  },
  "table": {
    "section_title": "セグメント別サマリー（FY2024）",
    "headers": ["セグメント", "売上高", "構成比", "CAGR", "営業利益率"],
    "rows": [
      {"name": "自動車事業", "revenue": "41,500", "share": "88.8%", "cagr": "+13.3%", "op_margin": "9.8%"},
      {"name": "金融事業", "revenue": "3,800", "share": "8.1%", "cagr": "+15.9%", "op_margin": "19.2%"},
      {"name": "その他事業", "revenue": "1,400", "share": "3.0%", "cagr": "+9.3%", "op_margin": "7.5%"}
    ]
  }
}
```

### JSONフィールド仕様

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `source` | string | 任意 | 出典テキスト（左下に表示） |
| `main_message` | string | 必須 | メインメッセージ。最大70文字、「〜すべき」で締める |
| `chart_title` | string | 任意 | チャートタイトル |
| `segments` | array | 必須 | セグメント定義の配列 |
| `segments[].name` | string | 必須 | セグメント名（テーブルのrows[].nameと一致させる） |
| `segments[].color` | string | 任意 | セグメント色（#RRGGBB形式）。未指定時は自動割り当て |
| `chart.section_title` | string | 任意 | 左側セクションタイトル。デフォルト「セグメント別売上高推移」 |
| `chart.unit_label` | string | 任意 | 単位表記（例：「（単位：十億円）」） |
| `chart.years` | array | 必須 | 年度ラベルの配列（例：["FY2020", ...]） |
| `chart.values` | object | 必須 | `{セグメント名: [年度ごとの値の配列]}`。 years と同じ長さの配列を各セグメントに指定 |
| `table.section_title` | string | 任意 | 右側セクションタイトル |
| `table.headers` | array | 必須 | テーブルのヘッダー配列 |
| `table.rows` | array | 必須 | テーブルの各行データ |
| `table.rows[].name` | string | 必須 | セグメント名（「セグメント」列） |
| `table.rows[].revenue` | string | 任意 | 売上高表記（例：「41,500」） |
| `table.rows[].share` | string | 任意 | 構成比表記（例：「88.8%」） |
| `table.rows[].cagr` | string | 任意 | 成長率表記（例：「+13.3%」） |
| `table.rows[].op_margin` | string | 任意 | 営業利益率表記（例：「9.8%」） |
| `table.rows[].op_profit` | string | 任意 | 営業利益表記（ヘッダーに「営業利益」がある場合） |

### ヘッダーキーマッピング

テーブルのヘッダー名から自動的に row のキーにマッピングされる：

| ヘッダー | マッピング先キー |
|---|---|
| 「セグメント」「事業セグメント」「セグメント名」「事業」 | `name` |
| 「売上」「売上高」「最新期売上」 | `revenue` |
| 「構成比」「シェア」 | `share` |
| 「CAGR」「成長率」 | `cagr` |
| 「利益率」「営業利益率」 | `op_margin` |
| 「営業利益」 | `op_profit` |

その他のヘッダーを使いたい場合は、row のキーとして `col0`, `col1`, ... を使う。

## 注意事項

- **セグメント数**: 3〜7セグメントを推奨。それ以上だと積み上げ棒が読みづらくなる
- **セグメント名の一貫性**: `segments[].name` / `chart.values` のキー / `table.rows[].name` はすべて一致させる
- **データラベル**: セグメントの値が極端に小さい（全体の3%未満など）場合、棒の中央に数字が収まらないことがある。その場合はラベルが棒をはみ出して表示されるが、PowerPointでの実際の表示は問題ないことが多い
- **非上場企業**: セグメント情報は通常開示されないため、本スキルは主に上場企業の対象会社・競合に適用する
- **競合比較**: 複数企業の事業ポートフォリオを比較したい場合は、各社ごとにスライドを生成する（本スキルは1社1スライド）

## 競合分析デッキでの使用例

`competitor-analyst-agent` オーケストレータースキルから呼び出される想定の位置づけ:

```
1. 対象会社プロファイル (customer-profile-pptx)
2. 対象会社 事業ポートフォリオ ← 本スキル
3. 市場環境 (market-environment-pptx)
4. 競合比較サマリー (competitor-summary-pptx)
5. 各競合プロファイル (customer-profile-pptx × N)
```

対象会社が多角化経営の場合、2枚目で事業ミックスを可視化することで、「どの事業領域で競合しているか」が明確になる。
