# バリューチェーン・ポジショニング・マトリクス PowerPoint ジェネレーター

業界全体のバリューチェーンを段階別に分解し、**自社のポジショニング（どの工程で何をするか）**を
1枚のスライドで多面的に可視化する、戦略コンサル定番のマトリクス分析スキル。

- 「どの工程を自社で担うか」
- 「どの工程を他社に任せるか」
- 「各工程の主なプレーヤーは誰か」
- 「どこが競合領域で、どこが協業候補か」

を1枚で示すことで、**資源配分・提携戦略・M&Aスコープの意思決定を支援**する。

---

## スライド構成

| セクション | 位置 | 内容 |
|---|---|---|
| **メインメッセージ** | 最上部 | 最大70文字。28pt Bold。「〜すべき」で締める |
| **チャートタイトル** | メインメッセージ直下 | 14pt。例「バリューチェーン・ポジショニング・マトリクス：○○業界」 |
| **シェブロン行** | 中央上部 | 5〜7段階のバリューチェーン段階を矢尻浅めのシェブロンで横並び |
| **行1：自社工程** | 下段1 | 各工程で自社が実施する度合いを記号で表現 |
| **行2：他社工程** | 下段2 | 各工程で外注・協力先に依存する度合いを記号で表現 |
| **行3：主なプレーヤー** | 下段3 | 複数列をまたぐバーで業界プレーヤーを表示 |
| **行4：競合・協力** | 下段4 | 各工程での競合／協力関係を記号で表現 |
| **出典** | 左下 | 情報ソースの記載（10pt グレー） |

### 行構成（4行固定）

画像の典型例に準拠した4行の分析軸。各行のラベルは自由にカスタマイズ可能。

| 行 | デフォルトラベル | タイプ | 内容 |
|---|---|---|---|
| 1 | 自社が実施する事業工程（収益源泉） | `symbols` | 記号セル |
| 2 | 他社に任せる工程（必要リソース） | `symbols` | 記号セル |
| 3 | 主なプレーヤー | `bars` | 複数列にまたがるバー |
| 4 | 競合：△／協力先：○（候補含） | `symbols` | 記号セル |

### 記号体系（5種類）

| 記号 | 意味（目安） | 色 |
|---|---|---|
| `◎` | 非常に強い／特に重要 | ネイビー `#1E3A5F` |
| `○` | 強い／実施 | 同上 |
| `△` | 部分的／中程度 | 同上 |
| `ー` | 該当なし／未実施 | 同上 |
| `""` (空文字) | セルを空白にする | — |

## 入力パターンと処理フロー

### パターンA：生データ・業界情報からマトリクスを組み立てる場合

**いきなりPowerPointを作成しない**

1. **Step 1: バリューチェーンを5〜7段階に分解**
2. **Step 2: 4行の観点ごとにセルの記号・バーを推定**
3. **Step 3: Markdownでユーザーに提示**し、確認・修正を求める
4. **Step 4: 承認後**、PowerPointを生成する

### パターンB：データが整理済みで入力された場合

1. **Step 1: 内容を確認**し、段階数・行数が揃っているか確認
2. **Step 2: 確認後、PowerPointを生成する**

### バリューチェーン分解の典型パターン

| 業界 | 典型的な段階 |
|---|---|
| 自動車・製造 | 企画・設計 → マーケティング → 開発・調達 → 製造 → 出荷・物流 → 販売 → アフターサービス |
| 食品・飲料 | 原料調達 → 加工・製造 → 包装 → 物流・配送 → 小売・販売 |
| ソフトウェア | 企画 → 開発 → 配布 → サポート → カスタマイズ |
| 医薬品 | R&D → 臨床試験 → 製造 → 流通 → 販売（MR） → 薬局・処方 |
| 建設 | 設計 → 資材調達 → 施工 → 検査 → アフターメンテナンス |

## JSONデータ仕様

`{{WORK_DIR}}/value_chain_matrix_data.json` に以下の形式で保存する：

```json
{
  "source": "出典：業界レポート、各社HP、ヒアリング結果（2024年度ベース）",
  "main_message": "対象会社は川下の販売・サービス領域に強みを持つが、製造領域での競争力強化と川上パートナーとの協業が急務である",
  "chart_title": "バリューチェーン・ポジショニング・マトリクス：自動車部品業界",
  "stages": [
    {"name": "企画・設計"},
    {"name": "マーケ\nティング"},
    {"name": "開発・調達"},
    {"name": "製造"},
    {"name": "出荷・物流"},
    {"name": "最終消費者\n販売"},
    {"name": "アフター\nサービス"}
  ],
  "rows": [
    {
      "label": "自社が実施する\n事業工程\n(収益源泉)",
      "type": "symbols",
      "cells": ["△", "○", "○", "◎", "ー", "ー", "ー"]
    },
    {
      "label": "他社に任せる\n工程\n(必要リソース)",
      "type": "symbols",
      "cells": ["○", "○", "○", "○", "○", "○", "○"]
    },
    {
      "label": "主なプレーヤー",
      "type": "bars",
      "bars": [
        {"label": "X社", "span_start": 0, "span_end": 3},
        {"label": "Y社／Z社", "span_start": 4, "span_end": 6}
      ]
    },
    {
      "label": "競合：△／\n協力先：○\n(候補含)",
      "type": "symbols",
      "cells": ["△", "△", "△", "△", "○", "○", "○"]
    }
  ]
}
```

### JSONフィールド仕様

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `source` | string | 任意 | 出典テキスト（左下10pt グレー） |
| `main_message` | string | 必須 | 28pt Bold、最大70文字推奨 |
| `chart_title` | string | 任意 | 14pt。デフォルト「バリューチェーン・ポジショニング・マトリクス」 |
| `stages` | array (5〜7) | 必須 | シェブロン段階の配列 |
| `stages[].name` | string | 必須 | 段階名。`\n` で改行を明示できる（推奨：長い名称は改行を入れる） |
| `rows` | array (4固定) | 必須 | 4行の分析行 |
| `rows[].label` | string | 必須 | 行ラベル。`\n` で改行可能 |
| `rows[].type` | string | 必須 | `"symbols"` または `"bars"` |
| `rows[].cells` | array | `symbols`型で必須 | 長さ = stages数。`◎` `○` `△` `ー` `""` のいずれか |
| `rows[].bars` | array | `bars`型で必須 | バー定義の配列 |
| `rows[].bars[].label` | string | 必須 | バー内表示テキスト |
| `rows[].bars[].span_start` | int | 必須 | 開始列インデックス（0始まり） |
| `rows[].bars[].span_end` | int | 必須 | 終了列インデックス（包含） |

### バーの指定例

```json
"bars": [
  {"label": "A社", "span_start": 0, "span_end": 2},   // 列 0-2 (3列分)
  {"label": "自社", "span_start": 3, "span_end": 3},  // 列 3 のみ
  {"label": "B社／C社", "span_start": 4, "span_end": 6}  // 列 4-6 (3列分)
]
```

- `span_start` = `span_end` の場合は1列のみのバー
- 複数バーは同じ行に共存可能（重なり不可）

### シェブロン形状

- `MSO_SHAPE.CHEVRON` を使用
- 矢尻深さ `adj=25000`（デフォルト50000より浅く、文字領域を広く確保）
- 各シェブロンは 0.12" オーバーラップして連続性を演出（最右除く）

### プレーヤーバー

- `MSO_SHAPE.RECTANGLE`（単純な塗りつぶし長方形、角丸なし）
- セル内パディング: 水平0.08"、垂直0.22"
- 色は全バー共通（ユーザーがPPT上で自由に変更可能）

## 注意事項

- **段階名は短く**: シェブロン形状の制約で、段階名は5〜6文字以内が推奨。それ以上は `\n` で改行
- **記号の意味付けはユーザー定義**: `◎` `○` `△` `ー` に業界標準の絶対的定義はない。行ラベルで意味を明示すること
- **プレーヤーバーの色は固定**: JSONでは色を指定できない。ユーザーがPPT上で必要に応じて変更すること
- **クラウド吹き出し（〈例〉…）は現状実装外**: 必要な場合はユーザーがPPT上で手動追加
- **4行構成は固定**: 3行や5行にしたい場合は本スキルでは対応不可。テンプレート側の修正が必要
- **全オブジェクトがネイティブ**: 画像貼り付けではないため、ユーザーがPPT上で色・サイズ・テキストを自由に編集可能

## BDDレポートでの配置位置（推奨）

```
1. エグゼクティブサマリー
2-8. 対象会社・市場・競合の各種分析
9. バリューチェーン・利益プール分析（value-chain-pptx）
10. バリューチェーン・ポジショニング・マトリクス ← 本スキル
11. 成長ドライバー分析
12. 推奨アクション
```

「利益プール分析」で**どこで儲けられるか**を示した後、本スキルで**自社はどこで戦うか**を示すと、
論理展開が綺麗に繋がる。
