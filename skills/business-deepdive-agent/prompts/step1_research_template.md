# Step 1 Web 検索クエリテンプレート — business-deepdive-agent

`business-deepdive-agent` の Step 1 で 5 論点それぞれについて WebSearch を実行する際の
クエリテンプレート。プレースホルダ `{parent_company}` / `{segment_name}` / `{industry}` を
Step 0 で受領した値で展開する。

各論点 **5-8 件** のクエリを実行し、結果を踏まえて Step 3 の data_NN_*.json を組み立てる。

---

## プレースホルダ

| 変数 | 例（タクシー事業） |
|---|---|
| `{parent_company}` | `第一交通産業` または `第一交通産業株式会社` |
| `{segment_name}` | `タクシー事業` |
| `{industry}` | `陸運業` または `タクシー業界`（任意） |

---

## 論点 1: 事業の概要は？（business-overview-pptx）

**目的**: 対象セグメントの事業内容・主要拠点・主要製品/サービス・従業員規模・セグメント業績推移を把握。

### クエリ例

1. `{parent_company} {segment_name} セグメント情報 有価証券報告書`
2. `{parent_company} {segment_name} 売上 営業利益 推移`
3. `{parent_company} {segment_name} 拠点 営業所`
4. `{parent_company} {segment_name} 従業員数 規模`
5. `{parent_company} 統合報告書 {segment_name}`
6. `{parent_company} 決算説明会 {segment_name}`
7. `{parent_company} {segment_name} 事業内容 公式`

### 取得したい情報

- セグメント名・事業内容（1-2 行）
- セグメント開始年（事業発足 or 親会社の事業開始年）
- 主要拠点（本社・主要工場・主要営業所・店舗数）
- 主要製品 / サービス
- セグメント従業員数（開示があれば）
- セグメント別売上高・営業利益率の過去 5-10 年推移（revenue_chart モード用）
- 開示が薄い場合は KPI（保有設備数・契約数等）を集める（kpi_cards モード用）

---

## 論点 2: その事業のビジネスモデルは？（business-model-pptx）

**目的**: 対象セグメントの収益構造・取引フロー・サプライヤー・顧客の関係を可視化。

### クエリ例

1. `{segment_name} ビジネスモデル 収益構造`
2. `{parent_company} {segment_name} IR 説明会`
3. `{segment_name} 業界 サプライヤー 仕入`
4. `{segment_name} 業界 取引フロー 商流`
5. `{parent_company} {segment_name} 主要仕入先`
6. `{segment_name} 業界 マネタイズ 収益モデル`
7. `{industry} 構造 プレイヤー`

### 取得したい情報

- 自社サービス名（中央配置用、最大 12 字）
- サプライヤー（左側 1-4 社）: 「何を提供」「対価」
- 顧客（右側 1-4 社）: 「何を提供」「対価」
- 各取引フローのテキスト（最大 15 字）
- ビジネスモデルの強み・リスク・機会（implications 3 点）

---

## 論点 3: その事業の差別化ポイントは？（value-chain-matrix-pptx）

**目的**: 業界バリューチェーン上で対象セグメントが「どこに強く、どこを他社に任せ、誰と競合・協業するか」を整理。

### クエリ例

1. `{segment_name} 業界 バリューチェーン 工程`
2. `{parent_company} {segment_name} 競合優位性 差別化`
3. `{parent_company} {segment_name} 競合 比較`
4. `{segment_name} 業界 主要プレイヤー シェア`
5. `{segment_name} 業界 提携 アライアンス`
6. `{industry} 競合 ポジショニング`
7. `{parent_company} {segment_name} 強み 弱み`
8. `{segment_name} 上下流 アウトソース`

### 取得したい情報

- 業界バリューチェーンの 5-7 段階（stages）
- 各段階で対象会社が実施する度合い（◎/○/△/ー）
- 各段階で他社に任せる度合い
- 主なプレーヤーのバー配置（span_start / span_end）
- 各段階での競合・協力関係

---

## 論点 4: その事業の顧客は誰か？（customer-profile-pptx）

**目的**: 対象セグメントの主要顧客（B2B なら企業、B2C なら顧客セグメント像）の概要・業績を把握。

### B2B の場合のクエリ例

1. `{parent_company} {segment_name} 主要販売先 取引先`
2. `{parent_company} 有価証券報告書 主要販売先`
3. `{segment_name} 業界 顧客 主要納入先`
4. `<顧客企業名> 売上 業績`（顧客が判明後）
5. `<顧客企業名> 事業内容 規模`

### B2C の場合のクエリ例

1. `{segment_name} 利用者 セグメント 構成`
2. `{segment_name} 顧客層 年代 世代`
3. `{segment_name} 需要 用途 シーン`
4. `{segment_name} 利用動向 統計`
5. `{industry} 顧客 行動 トレンド`

### 取得したい情報

- 主要顧客の商号・事業内容・本社・設立年・売上高・代表者・従業員数（B2B）
- または代表的顧客セグメントの規模・特徴・ニーズ（B2C）
- 顧客の業績推移（売上・営業利益率）5-10 年分（B2B、開示があれば）

---

## 論点 5: その事業の顧客は成長するか？（market-environment-pptx）

**目的**: 対象セグメントの **顧客側市場**（=「事業対象の顧客が属する市場」）の規模・成長率推移を把握。

会社全体の事業市場ではなく、**顧客が属する市場**であることに注意:
- タクシー事業 → 顧客市場 = 「観光・移動需要市場」「都市交通需要」
- 半導体製造装置 → 顧客市場 = 「半導体メーカーの設備投資額」「半導体出荷額」
- HR Tech SaaS → 顧客市場 = 「企業の人事関連支出」「労働市場規模」

### クエリ例

1. `<顧客市場名> 市場規模 推移 統計`
2. `<顧客市場名> 業界 規模 兆円`
3. `<顧客市場名> CAGR 成長率 予測`
4. `<顧客市場名> 矢野経済 富士経済 レポート`
5. `<顧客市場名> 経済産業省 国土交通省 統計`
6. `{segment_name} 業界 マクロ環境 トレンド`
7. `{segment_name} 需要 予測 中期`

### 取得したい情報

- 顧客市場の市場規模（年次）5-10 年分
- 市場のセグメント分解（2-3 系列、stacked_bars 用）
- CAGR（期間別）
- 折れ線で表現する比率指標（任意、構成比 / 浸透率 / 利益率等）
- 出典（官公庁統計 / 業界団体統計 / 調査会社レポート）

---

## クエリ実行のガイドライン

- 各クエリ間で重複検索を避ける（既出の URL は再 fetch しない）
- 1 論点あたり **5-8 件**（多すぎても情報過多、少なすぎると裏取り不足）
- 取得した数値は **出典 + 取得日** をメモする（fact-check しなくても data-availability に記録するため）
- 公式ソース優先順位: **有報 > 統合報告書 > 公式 HP > 業界団体 > 業界紙 > 調査会社レポート**
- 顧客市場（論点 5）は調査会社レポートが主要ソースになるが、無料公開部分のみ取得し有料部分は data-availability に「✗未取得」と記録

---

## 出力形式

各論点について Web 検索が完了したら、以下の構造で内部メモを作成（Step 3 の data_NN_*.json 組み立てに使う）:

```markdown
## 論点 1: 事業の概要

### 取得済情報
- セグメント名: タクシー事業
- 事業内容: 一般・観光タクシー、ハイヤー、貸切バスの運行
- 主要拠点: 全国 33 都道府県・119 拠点
- セグメント従業員数: 約 8,300 名（24/3 期）
- 業績推移（億円）: 2018: 540 / 2019: 555 / 2020: 460 / ...

### 出典
- 第一交通産業 有価証券報告書 第65期（2024年3月期）
- 公式HP「事業内容」ページ

### 未取得・data_gap
- セグメント別営業利益率の詳細内訳（推測値で代用）
```
