# 主要販売先売上高 PowerPoint ジェネレーター

BDD（ビジネスデュー・ディリジェンス）において、対象会社の主要販売先ごとの売上実績を
期別テーブルで横並びに表示するPowerPointスライドを生成するスキル。

**方式: PPTXネイティブテーブル（期ごとに独立テーブルを `add_table()` で横並び配置）**
- Playwright不要。全オブジェクトがPPT上で編集可能。
- company-history-pptx と同じネイティブテーブル方式。

---

## スライド構成

| セクション | 位置 | 内容 |
|---|---|---|
| **メインメッセージ** | 最上部 | 最大70文字。「〜すべき」で締める |
| **チャートタイトル** | メインメッセージ直下 | 「主要販売先からの完成工事売上高と割合」等 |
| **注記** | 右上（コンテンツエリア内） | 「オレンジ字：継続的な顧客（N期）」 |
| **期ヘッダー** | テーブル上部 | 期ラベル（太字・下線・中央寄せ）×N |
| **期別テーブル** | コンテンツエリア | 2〜5期の独立テーブルを横並び配置 |
| **出典** | 左下 | 情報ソースの記載 |

### レイアウト定数

| 要素 | 位置・サイズ |
|---|---|
| スライドサイズ | 13.33" × 7.50"（ワイドスクリーン） |
| コンテンツ左端 | 0.41" |
| コンテンツ上端 | 1.50" |
| コンテンツ幅 | 12.52" |
| テーブル間ギャップ | 0.15" |
| テーブル下端 | 6.90"（出典の上） |

### 各期テーブルの構成

| 列 | 比率 | 内容 | 備考 |
|---|---|---|---|
| # | 8% | 順位（1〜10） | その他は空欄、中央寄せ |
| 企業名 | 42% | 顧客企業名 | 継続顧客はオレンジ色＋太字、左寄せ |
| 売上高 | 30% | 売上高（千円等） | カンマ区切り、右寄せ |
| 割合 | 20% | 全体に占める割合（%） | 小数点1桁、右寄せ |

### 継続顧客のハイライト

- **全期間に出現する企業**をオレンジ色（#E67E00）＋太字で表示
- 「その他」は継続顧客の対象外
- 注記で「オレンジ字：継続的な顧客（N期）」と表示（Nは期数）

## 入力パターンと処理フロー

### パターンA：Excelファイルがアップロードされた場合

**いきなりPowerPointを作成しない**

1. **Step 1: Excelを読み取る**
   - 各シートのシート名から期のラベルを取得（例: "21/6期"）
   - 各シートから企業名と売上高を読み取る
   - 売上高の合計から各社の割合を算出
   - 売上高降順でランク付け、上位10社＋その他を集計

2. **Step 2: Markdownでユーザーに提示**し、確認・修正を求める

3. **Step 3: ユーザーの承認後**、PowerPointを生成する

### パターンB：データが整理済みで入力された場合

1. **Step 1: 内容を確認**し、期数・顧客データが揃っているか確認
2. **Step 2: 確認後、PowerPointを生成する**

## Step 1: Excelの読み取り

### 想定Excelフォーマット

**シート構成**: 期ごとに1シート（例: "21/6期", "22/6期", "23/6期"）

**各シートの列構成**:

| 列 | 内容 |
|---|---|
| 企業名（A列等） | 顧客企業名 |
| 売上高（B列等） | 売上高（数値） |

※ 列名が「企業名」「顧客名」「得意先」「取引先」等の場合は企業名列と判定
※ 列名が「売上高」「売上」「金額」「完成工事高」等の場合は売上高列と判定

### Excelの読み取り手順

```python
import openpyxl

wb = openpyxl.load_workbook("/mnt/user-data/uploads/xxxx.xlsx", data_only=True)
for sheet_name in wb.sheetnames:
    ws = wb[sheet_name]
    # ヘッダー行を特定し、企業名列と売上高列を判定
    # データ行を読み取り、売上高でソート
    # 上位10社＋その他を集計
```

### 集計ルール

1. 各期の全顧客の売上高を合計 → 期全体の売上高
2. 売上高降順でソート → 上位10社を抽出
3. 11位以下を「その他」として合算
4. 各社の割合 = 各社売上高 ÷ 期全体売上高 × 100（小数点1桁）

## Step 3: PowerPointの生成

### JSONデータ仕様

`/home/claude/sales_by_customer_data.json` に以下の形式で保存する：

```json
{
  "main_message": "主要販売先は○○社を中心に安定した取引基盤を有しており、顧客集中リスクを評価すべき",
  "chart_title": "主要販売先からの完成工事売上高と割合",
  "source": "対象会社提供資料",
  "subtitle_note": "",
  "continuous_label": "継続的な顧客",
  "continuous_threshold_label": "三期",
  "periods": [
    {
      "label": "21/6期",
      "unit": "千円",
      "customers": [
        {"rank": 1, "name": "ジップ", "revenue": 67830, "share": 18.1},
        {"rank": 2, "name": "大気社", "revenue": 67764, "share": 18.1},
        {"rank": -1, "name": "その他", "revenue": 7909, "share": 2.1}
      ]
    }
  ]
}
```

### JSONフィールド仕様

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `main_message` | string | 必須 | メインメッセージ。最大70文字、「〜すべき」で締める |
| `chart_title` | string | 任意 | チャートタイトル。デフォルト「主要販売先からの完成工事売上高と割合」 |
| `source` | string | 任意 | 出典テキスト（左下に表示） |
| `subtitle_note` | string | 任意 | 右上の注記。空欄なら自動生成（「オレンジ字：継続的な顧客（N期）」） |
| `continuous_label` | string | 任意 | 継続顧客のラベル。デフォルト「継続的な顧客」 |
| `continuous_threshold_label` | string | 任意 | 注記の閾値表示（例: "三期"）。空欄なら期数から自動生成 |
| `periods` | array | 必須 | 期別データの配列（2〜5要素） |
| `periods[].label` | string | 必須 | 期のラベル（例: "21/6期"） |
| `periods[].unit` | string | 任意 | 売上高の単位。デフォルト「千円」 |
| `periods[].customers` | array | 必須 | 顧客データの配列（上位10社＋その他） |
| `periods[].customers[].rank` | int | 必須 | 順位（1〜10）。その他は -1 |
| `periods[].customers[].name` | string | 必須 | 企業名 |
| `periods[].customers[].revenue` | number | 必須 | 売上高 |
| `periods[].customers[].share` | number | 必須 | 割合（%） |
